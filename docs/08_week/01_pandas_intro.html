8주차 1강: 판다스란 무엇인가? (Pandas Intro)

학습목표: 파이썬 데이터 분석의 심장, 판다스(Pandas)의 역할을 이해하고, 엑셀이나 데이터베이스(DB)와 어떤 점이 다른지 명확히 구분합니다.

8.1.1. 파이썬의 엑셀 (Excel for Python)

판다스(Pandas)는 파이썬에서 표(Table) 형태의 데이터를 다루기 위해 만들어진 라이브러리입니다. 금융 데이터 분석가였던 웨스 맥키니(Wes McKinney)가 “파이썬으로 엑셀처럼 편하게 데이터를 다룰 수 없을까?”라는 고민 끝에 만들었습니다.

  • 엑셀(Excel): 클릭으로 데이터를 보고 수정합니다. (직관적이지만 대용량 처리가 힘듦)
  • 판다스(Pandas): 코드로 데이터를 다룹니다. (자동화가 쉽고 대용량 처리 가능)



8.1.2. 왜 판다스를 배워야 할까요?

  1. 대용량 데이터 처리: 엑셀은 100만 행이 넘어가면 버벅거리지만, 판다스는 수천만 행도 거뜬합니다.
  2. 자동화(Automation): “매일 아침 지난달 매출 데이터를 정리해서 보고서 만들기” 같은 반복 작업을 코드 한 번 실행으로 끝낼 수 있습니다.
  3. 생태계(Ecosystem): 머신러닝(Scikit-learn), 딥러닝(TensorFlow/PyTorch), 시각화(Matplotlib) 등 모든 파이썬 라이브러리가 판다스 데이터프레임 형식을 기본으로 지원합니다.



8.1.3. 판다스 vs 데이터베이스 (RDBMS)

많은 분들이 “데이터를 표로 관리한다면, 데이터베이스(SQL)와 똑같은 거 아닌가요?”라고 묻습니다. 하지만 두 도구는 목적과 작동 방식이 완전히 다릅니다.

특징 판다스 (Pandas) 관계형 데이터베이스 (RDBMS)
주 무대 RAM (메모리) Disk (하드디스크)
속도 매우 빠름 (메모리 안에서 즉시 계산) 상대적으로 느림 (디스크 I/O 발생)
용량 RAM 용량에 의존 (보통 수 GB ~ 수십 GB) 디스크 용량만큼 저장 가능 (수 TB ~ PB)
목적 분석 (Analysis)
데이터를 이리저리 뜯어보고 실험하기		 기록 및 보관 (Transaction)
데이터를 안전하게 저장하고 무결성 유지
유연성 유연함 (Flexible)
데이터 타입이 섞여 있어도 OK		 엄격함 (Strict)
미리 정의된 스키마(Schema)를 따라야 함
작업 흐름 DB에서 데이터를 가져와서(Fetch) 분석 데이터를 저장하고(Store) 간단한 집계 수행

요약: 데이터베이스는 “거대한 도서관 서고”이고, 판다스는 “내 책상 위”입니다. 우리는 서고(DB)에서 필요한 책(Data)을 꺼내와서, 내 책상(Pandas)에 펼쳐놓고 분석 작업을 합니다.



8.1.4. 판다스 설치 및 시작

판다스는 보통 아나콘다(Anaconda)에 포함되어 있지만, 없다면 설치해야 합니다.

pip install pandas

불러올 때는 관례적으로 pd라는 별명을 사용합니다.

import pandas as pd
import numpy as np # 단짝 친구 넘파이도 같이 불러옵니다.



정리 (Summary)

이 강의에서 배운 핵심 내용을 요약해 봅시다.

  • [핵심 1]: 판다스(Pandas)는 파이썬에서 표 데이터를 다루는 표준 라이브러리이며, “코드로 다루는 엑셀”입니다.
  • [핵심 2]: 데이터베이스(RDBMS)가 저장과 관리에 특화되었다면, 판다스는 분석과 가공에 특화된 인메모리(RAM) 도구입니다.
  • [핵심 3]: import pandas as pd로 불러오며, 넘파이(Numpy)를 기반으로 만들어져서 매우 빠르고 강력합니다.
