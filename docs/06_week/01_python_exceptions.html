6주차 1강: 예외 처리 (Exception Handling)

학습목표: 프로그램 실행 중 발생하는 에러(예외)를 우아하게 처리하여, 프로그램이 갑자기 멈추는 것을 방지하는 방법을 배웁니다.

6.1.1. 에러가 나면 멈춘다?

일반적으로 프로그램은 에러가 발생하면 그 즉시 종료(Crash)됩니다. 하지만 중요한 서버나 게임이 에러 하나 때문에 꺼지면 안 되겠죠?

이때 사용하는 것이 try-except 구문입니다.



6.1.2. 기본 구조: try-except

“일단 시도해봐(try). 만약 에러가 나면(except) 이렇게 해.”

try:
    # 에러가 발생할 가능성이 있는 코드
    number = 10 / 0
except ZeroDivisionError:
    # 에러가 발생했을 때 실행할 코드
    print("0으로 나눌 수 없습니다! (프로그램은 계속 실행됩니다)")



[실행 결과]

0으로 나눌 수 없습니다! (프로그램은 계속 실행됩니다)

(빨간 에러 메시지 대신, 우리가 정한 문구가 출력되고 프로그램은 죽지 않습니다.)



6.1.3. 다양한 에러 잡기

여러 종류의 에러를 각각 다르게 처리할 수도 있습니다.

try:
    arr = [1, 2, 3]
    print(arr[5]) # 없는 인덱스 접근
except IndexError:
    print("배열의 범위를 벗어났어요!")
except Exception as e:
    print(f"알 수 없는 에러가 발생했습니다: {e}")



정리 (Summary)

이 강의에서 배운 핵심 내용을 요약해 봅시다.

  • [핵심 1]: 예외 처리(Exception Handling)는 프로그램이 에러로 인해 비정상 종료되는 것을 막고 안전하게 처리하는 기법입니다.
  • [핵심 2]: try-except 구문을 기본으로, 성공 시 else, 무조건 실행되는 finally를 조합해 사용합니다.
  • [핵심 3]: raise를 사용하면 필요할 때 강제로 에러를 발생시킬 수 있습니다. (신호 보내기)
