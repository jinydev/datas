# 9주차 3강: 정밀 타격 loc와 iloc (Indexing)

판다스에서 가장 헷갈리는 것이 `loc`와 `iloc`입니다. 이 둘의 차이만 알면 인덱싱은 끝입니다.

## 9.3.1. 이름으로 찾기: `loc` (Label location)

> **규칙**: `df.loc[행이름, 열이름]`
> "철수(행)의 영어(열) 점수는?"

```python
# 인덱스를 이름으로 설정
df = df.set_index('이름')

# 철수의 영어 점수
score = df.loc['철수', '영어']
print("철수 영어:", score) # 100

# 영희부터 민수까지, 국어 점수만
print(df.loc['영희':'민수', '국어'])
```
> **주의**: `loc`의 슬라이싱은 **끝 이름도 포함**합니다! ('민수' 포함)


<br>

## 9.3.2. 위치(번호)로 찾기: `iloc` (Integer location)

> **규칙**: `df.iloc[행번호, 열번호]`
> "0번째 줄의, 1번째 칸 값은?"

```python
# 0행(철수) 1열(영어) (국어가 0열, 영어가 1열이라고 가정 시)
score = df.iloc[0, 1]
print(score)
```
> **주의**: `iloc`의 슬라이싱은 리스트처럼 **끝 번호 미포함**입니다.

## 9.3.3. 비교 요약

| 구분 | `loc` | `iloc` |
|:---:|:---|:---|
| **기준** | **이름 (Label)** | **순서 번호 (Index)** |
| **슬라이싱** | 끝 포함 (`'A':'C'` -> A, B, C) | 끝 미포함 (`0:3` -> 0, 1, 2) |
| **비유** | "강남구 역삼동" (주소) | "위도 37.5, 경도 127.0" (GPS 좌표) |
