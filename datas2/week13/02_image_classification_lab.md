# 02. Lab: 나만의 이미지 분류기 (Image Classification)

## 1. 실험실 목표
인터넷에서 구한 실제 이미지(예: 피자/스테이크/스시 음식 사진)를 분류하는 모델을 만듭니다. 폴더 구조로 데이터를 정리하고 읽어오는 실무 파이프라인을 익힙니다.

## 2. 데이터 수집 및 폴더 구조
데이터는 `train/` 폴더와 `test/` 폴더 아래에 각 클래스 이름(예: `cat`, `dog`)으로 폴더가 나뉘어 있어야 합니다.
이 구조만 맞추면 케라스가 알아서 라벨을 붙여줍니다.

## 3. `ImageDataGenerator`와 `flow_from_directory`
이미지 파일을 하나씩 `imread`로 읽어오면 메모리가 터집니다.
제너레이터(Generator)를 통해 배치 단위(32장씩)로 그때그때 읽어오는 효율적인 방식을 씁니다.
`rescale=1./255`로 정규화도 동시에 수행합니다.

## 4. 모델 설계: CNN vs 전이 학습
데이터가 적다면(클래스당 100장 미만) 처음부터 CNN을 학습시키는 것보다 전이 학습이 압도적으로 유리합니다. MobileNetV2 같은 가벼운 모델을 사용합니다.

## 5. MobileNetV2 세팅
ImageNet 가중치를 가져오되, 맨 위(Top) 분류 층은 뺍니다. 입력 크기는 (224, 224, 3)으로 설정합니다.

## 6. 특징 추출 층(Base Model) 동결 (Freezing)
 `base_model.trainable = False`.
이미지를 보는 눈(특징 추출 능력)은 그대로 쓰고, 마지막 판단(분류) 능력만 새로 가르칩니다. 학습 속도가 매우 빠릅니다.

## 7. 분류기(Classifier) 부착
`GlobalAveragePooling2D` -> `Dropout` -> `Dense(3, softmax)` (피자, 스테이크, 스시).
 GAP 층을 쓰면 파라미터가 적어 모델이 가벼워집니다.

## 8. 모델 컴파일 및 학습
옵티마이저는 `Adam`을 쓰되, 미세 조정(Fine Tuning) 시에는 학습률(Learning Rate)을 아주 낮게(1e-5) 잡아야 기존 지식이 깨지지 않습니다.

## 9. 미세 조정 (Fine Tuning) - 심화
초기 학습(동결 상태에서 5 Epoch)이 끝나면, 기저 모델의 상위 층 일부만 동결을 풀고(`trainable=True`) 전체를 다시 살짝 학습시킵니다. 정확도를 1~2% 더 끌어올릴 수 있습니다.

## 10. 테스트 이미지 예측
인터넷에서 아무 음식 사진이나 다운받아 테스트해 봅니다.
`load_img` -> `img_to_array` -> `expand_dims` (배치 차원 추가) 과정을 거쳐야 모델에 넣을 수 있습니다.

## 11. 오분류 분석
피자를 피자라 부르지 못하는 경우를 봅니다. 토핑이 너무 많거나 모양이 이상하면 틀릴 수 있습니다.

## 12. 모델 저장 (`.h5` vs `.tflite`)
PC에서 쓸 모델은 `.h5`로 저장하고, 안드로이드 앱에 넣을 모델은 `.tflite` (TensorFlow Lite)로 변환하여 저장합니다. 모바일 배포의 기초입니다.

## 13. Grad-CAM 시각화 - 심화
모델이 피자 사진의 "페퍼로니" 부분을 보고 피자라고 판단했는지 열지도(Heatmap)로 그려봅니다. 설명 가능한 AI의 영역입니다.

## 14. 데이터 불균형 처리
스테이크 사진은 1000장인데 스시 사진은 100장이라면?
`class_weights` 옵션을 주어 소수 클래스에 가중치를 더 둡니다.

## 15. 실시간 웹캠 인식 (OpenCV 연동) - 선택 사항
노트북 웹캠으로 물체를 비추면 실시간으로 라벨을 띄워주는 코드를 작성해 봅니다. OpenCV의 `VideoCapture`와 연동합니다.

## 16. Teachable Machine (구글) 소개
코딩 없이 웹 브라우저에서 전이 학습을 시켜주는 도구입니다. 원리가 우리가 짠 코드와 똑같다는 것을 이해합니다.

## 17. Python 문법 집중: `os.walk`
폴더 내의 이미지가 몇 장인지 세거나 파일 경로를 가져올 때 사용하는 파일 시스템 탐색 함수입니다.

## 18. 다양한 전이 학습 모델들
*   **InceptionV3**: 구글의 자존심.
*   **EfficientNetB0~B7**: 성능 끝판왕. 숫자가 클수록 무겁고 성능 좋음.

## 19. 과제
자신의 얼굴과 친구의 얼굴을 분류하는 모델을 만들어 보십시오. 셀카 20장씩이면 충분합니다.
