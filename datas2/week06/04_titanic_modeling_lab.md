# 04. Lab: 타이타닉 생존 예측 챌린지 (최종 실습)

## 1. 실험실 목표
지금까지 배운 모든 것(전처리, 시각화, 파이프라인, 모델링, 평가, 튜닝)을 총동원하여 타이타닉 생존 예측 모델의 **최고 점수**에 도전합니다.

## 2. 데이터 준비
*   `train.csv`: 학습 및 검증용
*   `test.csv`: 최종 제출용 (정답지 없음)
*   외부 데이터 사용 금지.

## 3. 피처 엔지니어링 전략 (복습)
*   `Title` 그룹화 ('Mr', 'Mrs', 'Miss', 'Master', 'Rare')
*   `FamilySize` 및 `IsSolo` 생성
*   `Age` 및 `Fare`의 구간화(Binning) 또는 스케일링

## 4. 모델 선택 (알고리즘 월드컵)
여러 모델을 동시에 돌려보고 승자를 정합니다.
*   Logistic Regression: 베이스라인
*   Decision Tree: 기본 트리
*   Random Forest: 강력한 앙상블 (배깅)
*   Gradient Boosting: 더 강력한 앙상블 (부스팅)

## 5. 파이프라인 구축
전처리와 모델을 묶어서 `Pipeline` 객체로 만듭니다. 이를 통해 교차 검증 시 데이터 누수를 원천 차단합니다.

## 6. 교차 검증 (Cross Validation) 수행
단순 `train_test_split`이 아닌 5-Fold 또는 10-Fold CV를 사용하여 모델의 평균적인 성능을 측정합니다. 이것이 우리의 '리더보드' 점수가 됩니다.

## 7. 하이퍼파라미터 튜닝 (GridSearchCV)
랜덤 포레스트의 `n_estimators`(나무 개수)와 `max_depth`를 조절하여 최적의 조합을 찾습니다. 컴퓨터가 밤새 일하게 하십시오.

## 8. 앙상블 (Voting Classifier)
단일 모델로 한계가 느껴진다면? 로지스틱, 랜덤 포레스트, SVM을 섞어서 투표를 붙이는 '보팅' 기법을 사용합니다.

## 9. 오차 분석 (Error Analysis)
모델이 틀린 데이터만 모아서 분석해 봅니다.
"왜 우리 모델은 유독 '3등석 성인 남성'을 틀릴까?"
이유를 찾아내고 피처를 수정합니다.

## 10. 정밀도 vs 재현율 조정
만약 실제 구조 상황이라면 재현율(놓치지 않는 것)을 높여야 합니다. `predict_proba()`의 임계값을 0.5에서 0.3으로 낮추어 봅니다.

## 11. 최종 모델 선정
교차 검증 평균 점수와 편차(Standard Deviation)를 고려하여 가장 안정적이고 성능 좋은 모델을 선택합니다.

## 12. 전체 데이터 재학습 (Retrain)
검증이 끝났다면, 훈련 데이터 전체(Validation set 포함)를 사용하여 선택된 모델을 다시 학습시킵니다. 데이터가 많을수록 성능은 좋아집니다.

## 13. 테스트 데이터 예측
`test.csv`에 대해 `predict`를 수행합니다.

## 14. 제출 파일 생성 (`submission.csv`)
Kaggle이나 데이콘 형식에 맞춰 `PassengerId,Survived` 컬럼만 남긴 CSV를 만듭니다.

## 15. 리포트 작성: 우리는 무엇을 배웠는가?
어떤 피처가 가장 중요했는지, 모델별 장단점은 무엇이었는지 정리합니다.

## 16. 모델 해석 (SHAP Values) - 심화
(선택 사항) SHAP 라이브러리를 사용하면 복잡한 앙상블 모델도 "이 승객은 나이 때문에 생존 확률이 10% 깎였고, 성별 때문에 30% 올랐다"고 설명할 수 있습니다.

## 17. 실무적 고려사항
이 모델을 실제 선박 API에 배포한다면? 응답 속도(Latency)와 모델 크기(Size)도 고려해야 합니다. 랜덤 포레스트는 무거울 수 있습니다.

## 18. 윤리적 AI 생각하기
이 모델은 "여성을 우선 구조하라"는 과거의 편향을 학습했습니다. 이것이 현대의 재난 구조 알고리즘으로 적합한지 토론해 봅니다.

## 19. 전체 코드 리팩토링
지저분한 노트북 코드를 `titanic_predictor.py`라는 깔끔한 클래스 구조로 정리합니다.

## 20. 대장정의 마무리
타이타닉 프로젝트가 끝났습니다. 이제 여러분은 데이터를 다루는 법, 전처리하는 법, 모델을 만드는 법을 모두 아는 데이터 분석가입니다.
