# 01. 대단원의 막 (Final Project 가이드)

## 1. 프로젝트 개요
한 학기 동안 배운 모든 지식(데이터 수집, 전처리, 시각화, 머신러닝/딥러닝)을 총동원하여, 자신만의 데이터 분석 프로젝트를 수행합니다.
"데이터로 세상을 이해하고, 문제를 해결하는 데이터 사이언티스트"가 되는 마지막 관문입니다.

## 2. 주제 선정 (Topic Selection)
가장 중요한 단계입니다. "내가 좋아하는 것"이나 "해결하고 싶은 문제"에서 시작하세요.
*   **좋은 주제**: 데이터가 구하기 쉽고, 해결하려는 목표(Target)가 명확한 것. (예: "서울시 따릉이 수요 예측", "리그오브레전드 승패 예측")
*   **나쁜 주제**: 데이터가 없거나, 너무 거창한 것. (예: "주식 시장 완벽 예측", "인류의 행복도 분석")

## 3. 데이터 수집 (Data Collection)
*   **Kaggle / Dacon**: 가장 쉽고 품질 좋은 데이터를 구할 수 있습니다.
*   **공공데이터포털**: 한국 데이터(부동산, 날씨, 교통)를 얻기 좋습니다.
*   **Web Scraping**: `BeautifulSoup`, `Selenium` 등을 이용해 직접 긁어옵니다. (법적 문제 주의)

## 4. 데이터 탐색 (EDA)
데이터를 눈으로 확인하는 과정입니다.
*   데이터의 크기(Shape), 타입(info), 결측치(isnull) 확인.
*   `describe()`로 기초 통계량 확인.
*   히스토그램, 산점도, 상관관계 히트맵 시각화.
*   **인사이트 도출**: "주말에는 평일보다 따릉이 이용량이 2배 많구나!"

## 5. 데이터 전처리 (Preprocessing)
모델이 학습할 수 있게 데이터를 다듬습니다.
*   **결측치 처리**: 평균으로 채우기, 삭제하기, 혹은 예측해서 채우기.
*   **이상치 처리**: IQR 방식으로 너무 튀는 값 제거.
*   **인코딩**: 범주형 데이터(문자)를 숫자(원-핫/라벨)로 변환.
*   **스케일링**: 수치형 데이터의 단위를 맞춤 (`StandardScaler`).

## 6. 피처 엔지니어링 (Feature Engineering)
모델의 성능을 결정짓는 핵심입니다. 도메인 지식을 활용해 새로운 변수를 만듭니다.
*   "탑승 시간"과 "하차 시간" -> "이동 시간" 변수 생성.
*   "날짜" -> "요일", "공휴일 여부" 변수 생성.

## 7. 모델 선택 (Model Selection)
문제의 종류에 맞는 모델을 고릅니다.
*   **정형 데이터 분류**: Random Forest, XGBoost, LightGBM (최강자들).
*   **이미지**: CNN (VGG, ResNet, EfficientNet).
*   **텍스트**: LSTM, BERT (Hugging Face).
*   **시계열**: ARIMA, Prophet, LSTM.

## 8. 모델 학습 및 튜닝 (Training & Tuning)
*   `train_test_split`으로 검증 세트 분리 필수.
*   교차 검증(`cross_val_score`)으로 일반화 성능 확인.
*   `GridSearchCV`나 `Optuna`로 하이퍼파라미터 최적화.

## 9. 모델 평가 (Evaluation)
*   **회귀**: MSE, RMSE, R2 Score.
*   **분류**: Accuracy, Precision, Recall, F1-Score, ROC-AUC.
단순 정확도만 보지 말고, 오차 행렬(Confusion Matrix)을 통해 "무엇을 틀렸는지" 분석해야 합니다.

## 10. 결과 해석 및 시각화
"정확도 95%입니다"로 끝내지 마세요.
SHAP, Permutation Importance 등을 이용해 "어떤 변수가 예측에 가장 중요했는지" 설명해야 합니다.
"집값을 예측할 때는 평수보다 지하철 거리가 더 중요했습니다."

## 11. 결론 및 제언
분석 결과를 바탕으로 비즈니스 액션을 제안합니다.
"따라서 비가 오는 주말에는 따릉이 배치 수를 30% 줄여야 합니다."

## 12. 보고서 작성 (Reporting)
코드는 깨끗하게 정리하고(주석 필수), PPT나 PDF로 결과물을 요약합니다. 면접관은 당신의 코드가 아니라 당신의 논리 구조를 봅니다.

## 13. Streamlit 배포 (선택 사항)
`Streamlit` 라이브러리를 쓰면 파이썬 코드 몇 줄로 멋진 웹 대시보드를 만들 수 있습니다. 내 모델을 웹에 올려서 친구들에게 자랑해 보세요.

## 14. 협업 툴 (Git & GitHub)
팀 프로젝트라면 깃허브를 통해 코드를 공유하고 버전 관리하는 법을 익혀야 합니다. 커밋 메시지는  의미 있게 작성하세요.

## 15. 실패를 두려워하지 마세요
모델 성능이 안 나올 수도 있습니다. 중요한 것은 "왜 성능이 안 나왔는지" 분석하고, "어떻게 개선하려고 시도했는지" 기록하는 과정입니다. 그것이 경험입니다.

## 16. 윤리적 고려
개인정보 보호, 편향된 데이터(성차별, 인종차별) 문제에 대해 고민해 보았나요? AI 윤리는 이제 선택이 아닌 필수입니다.

## 17. 참고 문헌 표기
사용한 데이터의 출처, 참고한 블로그나 논문을 반드시 명시합니다. 표절은 절대 금물입니다.

## 18. 자주 하는 실수 Check
*   데이터 누수(Data Leakage): 테스트 데이터를 학습에 써버림.
*   과대적합(Overfitting): 리더보드 점수는 높은데 실제로는 엉망.
*   설명 부족: 그래프만 띡 그려놓고 해석이 없음.

## 19. 최종 점검
*   코드는 에러 없이 처음부터 끝까지(`Run All`) 실행되는가?
*   파일 경로는 상대 경로 (`./data`)로 설정했는가?
*   발표 시간(5분/10분)을 맞췄는가?
