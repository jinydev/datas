# 01. 수치의 예측 (회귀 분석 기초)

## 1. 회귀(Regression)의 유래
프랜시스 골턴이 "키가 큰 부모의 자식은 부모보다 작아지고, 작은 부모의 자식은 커져서 전체 평균으로 돌아간다(Regress)"는 현상을 발견한 데서 유래했습니다. 지금은 '수치를 예측하는 모든 문제'를 통칭합니다.

## 2. 분류 vs 회귀
*   **분류(Classification)**: 생존/사망, 고양이/강아지 (범주형)
*   **회귀(Regression)**: 내일의 주가, 아파트 가격, 신생아 몸무게 (연속형 숫자)

## 3. 선형 회귀 (Linear Regression)
데이터의 경향성을 가장 잘 설명하는 **하나의 직선**을 긋는 것입니다.
$y = wx + b$
*   $w$ (Weight): 기울기 (가중치)
*   $b$ (Bias): 절편 (편향)

## 4. 단순 선형 회귀 (Simple Linear Regression)
변수($x$)가 하나인 경우입니다. "공부 시간($x$)이 늘어나면 성적($y$)이 오른다."

## 5. 다중 선형 회귀 (Multiple Linear Regression)
변수가 여러 개인 경우입니다. "공부 시간($x_1$), 아이큐($x_2$), 부모님 잔소리($x_3$)가 성적($y$)에 영향을 미친다."
$y = w_1x_1 + w_2x_2 + ... + b$

## 6. 손실 함수 (Loss Function) - MSE
직선이 데이터를 얼마나 잘 설명하는지(또는 얼마나 틀렸는지) 측정하는 자입니다.
주로 평균 제곱 오차(MSE: Mean Squared Error)를 사용합니다. 오차를 제곱하므로 많이 틀릴수록 페널티가 커집니다.

## 7. 학습의 원리: 경사 하강법 (Gradient Descent)
오차(MSE)를 최소화하는 $w$와 $b$를 찾는 과정입니다. 산 정상에서 공을 굴렸을 때 가장 낮은 골짜기로 내려가는 원리를 수학적으로 구현한 것입니다.

## 8. 사이킷런의 `LinearRegression`
경사 하강법 대신 수학적 공식(정규 방정식)으로 한 방에 정답을 찾는 OLS(Ordinary Least Squares) 방식을 사용합니다. 데이터가 아주 크지 않을 때 효율적입니다.

## 9. 상관관계와 회귀분석의 차이
상관관계는 "두 변수가 관련이 있다($r=0.8$)"는 상태만 말해주지만, 회귀분석은 "온도가 1도 오르면 아이스크림 판매량은 50개 늘어난다($w=50$)"는 인과적 관계와 예측값을 줍니다.

## 10. 결정 계수 ($R^2$ Score)
회귀 모델의 성적표입니다. "이 직선이 데이터 분산의 몇 %를 설명하는가?"
1에 가까울수록 완벽한 예측입니다.

## 11. 실습: 키와 몸무게 데이터
가상의 키($x$)와 몸무게($y$) 데이터를 생성하여, $x$가 주어졌을 때 $y$를 맞추는 모델을 만듭니다.

## 12. 회귀 계수 해석 (Coefficient Interpretation)
모델 학습 후 `model.coef_`를 찍어보면 기울기가 나옵니다. 이 숫자가 곧 변수의 영향력입니다.

## 13. 절편의 의미 (Intercept)
`model.intercept_`는 $x$가 0일 때의 기본값입니다.

## 14. 다항 회귀 (Polynomial Regression)
데이터가 직선이 아니라 곡선 형태라면?
$x^2$ 항을 추가하여 2차 함수($y=ax^2+bx+c$) 형태로 모델링합니다. 이것도 넓은 의미의 선형 회귀입니다.

## 15. 과대적합 주의
차수(Degree)를 너무 높이면 훈련 데이터의 모든 점을 지나가는 꾸불꾸불한 선이 만들어집니다. (새로운 데이터 예측 실패)

## 16. 규제 선형 모델 (Ridge, Lasso)
과대적합을 막기 위해 $w$값이 너무 커지지 않도록 족쇄(Penalty)를 채운 모델들입니다. L1규제(Lasso), L2규제(Ridge).

## 17. 실무적 고려: 외삽(Extrapolation)의 위험
훈련 데이터의 범위(키 150~190)를 벗어난 데이터(키 250)가 들어오면, 선형 모델은 엉뚱한 예측을 할 수 있습니다.

## 18. Python 문법 집중: `reshape`
사이킷런의 입력은 항상 2차원 매트릭스여야 합니다. 1차원 배열(`[1, 2, 3]`)을 `[[1], [2], [3]]`으로 바꿔주는 `reshape(-1, 1)` 테크닉이 필수입니다.

## 19. 잔차 분석 (Residual Analysis)
예측값과 실제값의 차이(잔차)가 특정한 패턴 없이 무작위로 퍼져 있어야 좋은 모델입니다.

## 20. 결론
세상의 많은 현상은 선형적(Linear)입니다. 가장 단순하지만 가장 강력한 예측 도구인 선형 회귀를 마스터합니다.
