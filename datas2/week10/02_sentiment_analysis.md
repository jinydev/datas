# 02. Lab: 영화 리뷰 감성 분석 (Sentiment Analysis)

## 1. 실험실 목표
영화 리뷰 텍스트("이 영화 정말 재밌다" vs "시간 낭비였다")를 읽고, 긍정(Positive)인지 부정(Negative)인지 맞추는 분류 모델을 만듭니다.

## 2. 데이터 준비 (Naver Movie Ratings)
한국어 실습을 위해 네이버 영화 리뷰 데이터셋(`ratings_train.txt`)을 사용합니다. 15만 개의 리뷰가 있습니다.
(colab 등에서는 `wget`으로 다운로드)

## 3. 데이터 로드 및 탐색
판다스로 열어보면 `id`, `document`(리뷰), `label`(1:긍정, 0:부정) 컬럼이 보입니다.
결측치(내용 없음)와 중복 데이터(매크로)를 제거합니다.

## 4. 한국어 전처리 (KoNLPy Okt)
영어와 달리 단순히 띄어쓰기로 자르면 "영화가", "영화를", "영화는"이 다 다른 단어가 됩니다.
`Okt.morphs(stem=True)`를 사용하여 "영화"라는 어간을 추출하고 조사를 버립니다. 시간이 꽤 걸리는 작업입니다.

## 5. 불용어(Stopwords) 정의
"아", "휴", "아이구", "을", "를" 같은 의미 없는 감탄사와 조사를 리스트로 정의해 제거합니다.

## 6. 벡터화 (TF-IDF)
전처리된 텍스트 리스트를 `TfidfVectorizer`에 넣어 숫자 매트릭스로 변환합니다. `min_df=3` 등으로 너무 희귀한 단어는 배제하여 차원을 줄입니다.

## 7. 가볍고 빠른 모델: 로지스틱 회귀
텍스트 분류의 베이스라인으로 로지스틱 회귀가 아주 훌륭합니다. 수만 차원의 희소 데이터에서도 잘 작동합니다.

## 8. 모델 학습 및 평가
훈련 데이터로 학습하고, 테스트 데이터로 정확도를 봅니다. 85% 정도 나오면 훌륭합니다.

## 9. 모델이 배운 단어들 (Coefficient)
로지스틱 회귀의 계수(`coef_`)를 뜯어봅니다.
양수 값이 큰 단어: "최고", "감동", "명작" (긍정)
음수 값이 큰 단어: "최악", "쓰레기", "졸작" (부정)
모델이 상식과 부합하게 학습했는지 검증합니다.

## 10. 직접 쓴 리뷰로 테스트
"이 영화 진짜 시간 아깝다 핵노잼 ㅠㅠ" 같은 문장을 넣어보고 모델이 0(부정)을 내놓는지 확인하는 재미있는 시간입니다.

## 11. 파이프라인 최적화
전처리와 모형 학습 과정을 `TfidfVectorizer` -> `LogisticRegression` 파이프라인으로 연결하여 관리합니다.

## 12. N-gram 효과 확인
단어 하나(Unigram)만 썼을 때와 두 단어(Bigram)를 섞어 썼을 때("재미 없다" vs "재미", "없다") 성능 차이를 비교합니다.

## 13. 한계점
"재밌는 줄 알았는데 아니네" 같은 반어법이나, "배우 연기는 좋은데 스토리는 망" 같은 복합 감정은 단순 Bag of Words 방식으로는 파악하기 어렵습니다. LSTM/BERT 같은 딥러닝이 필요한 이유입니다.

## 14. 딥러닝 맛보기 (LSTM 개념)
단어의 순서(Sequence) 정보를 기억하는 RNN/LSTM 모델이 왜 텍스트 처리에 강한지 개념만 짚고 넘어갑니다.

## 15. 추가 실습: 스팸 메일 분류
긍/부정뿐만 아니라, 스팸/햄 분류도 똑같은 원리입니다. 영문 데이터(SMS Spam Collection)로도 쉽게 실습 가능합니다.

## 16. 실무 적용: CS 자동 분류
고객센터 게시판 글을 읽고 "배송 문의", "환불 문의", "제품 불량" 카테고리로 자동 분류하는 시스템도 이 원리로 만듭니다.

## 17. Python 효율화: `pickle`
오래 걸리는 형태소 분석 결과를 매번 다시 돌리지 말고, `pickle`이나 `joblib`으로 저장해 두는 팁을 배웁니다.

## 18. 시각화: 긍정/부정 워드클라우드
긍정 리뷰만 모아서, 부정 리뷰만 모아서 각각 워드클라우드를 그려 봅니다. 등장하는 단어가 확연히 다릅니다.

## 19. 과제
자신이 스크래핑한 뉴스 기사나 댓글 데이터를 가져와서 감성 분석을 해보세요.
