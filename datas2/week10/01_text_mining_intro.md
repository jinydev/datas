# 01. 문자를 숫자로 (텍스트 마이닝 기초)

## 1. 자연어 처리(NLP)란?
컴퓨터가 인간의 언어(Natural Language)를 이해하고 생성하게 만드는 인공지능 분야입니다. 번역기, 챗봇, 스팸 필터가 여기에 속합니다.

## 2. 텍스트 데이터의 특징
비정형(Unstructured) 데이터입니다. 길이가 제각각이고, 오타가 있고, 문맥에 따라 의미가 달라집니다. 머신러닝 모델에 넣으려면 고정된 길이의 벡터(숫자 배열)로 변환해야 합니다.

## 3. 토큰화 (Tokenization)
문장을 의미 있는 조각(Token)으로 자르는 과정입니다.
*   **영어**: 띄어찍기(white space) 기준으로 잘라도 꽤 괜찮습니다. (NLTK, SpaCy)
*   **한국어**: 띄어쓰기가 복잡하므로 형태소 분석기(KoNLPy - Okt, Mecab)가 필수입니다. ("아버지가방에" -> "아버지", "가", "방", "에")

## 4. 정제(Cleaning)와 불용어(Stopwords)
기계가 학습하는 데 방해되는 노이즈를 제거합니다.
*   특수문자, HTML 태그 제거.
*   "은, 는, 이, 가, the, a, is" 같이 빈번하지만 의미 없는 단어(불용어) 제거.

## 5. 어간 추출(Stemming)과 표제어 추출(Lemmatization)
단어의 원형을 찾습니다.
*   goes, going, went -> go
단어 수를 줄여서 차원의 저주를 막습니다.

## 6. Bag of Words (BoW)
단어의 순서는 무시하고, 단순히 단어가 몇 번 등장했는지 세어서 벡터를 만듭니다.
"바나나 먹고 싶다" -> `{'바나나': 1, '먹고': 1, '싶다': 1}`.

## 7. 카운트 기반 벡터화 (CountVectorizer)
사이킷런 도구입니다. 전체 문서 집합에서 단어 사전을 만들고, 각 문서를 단어 등장 횟수 벡터로 변환합니다. `[0, 1, 1, 0, 3 ...]`.

## 8. TF-IDF (단어 빈도-역문서 빈도)
단순히 많이 나온 단어가 중요할까요? "오늘"이라는 단어는 모든 일기에 나오므로 중요하지 않습니다.
특정 문서에는 많이 나오지만(TF 높음), 전체 문서에는 적게 나오는(IDF 높음) 단어에 가중치를 주는 기법입니다. 핵심 키워드 추출에 쓰입니다.

## 9. 문서 단어 행렬 (DTM)
행(Row)은 문서, 열(Column)은 단어로 이루어진 거대한 행렬입니다. 대부분의 값이 0인 희소 행렬(Sparse Matrix)이 됩니다.

## 10. 워드클라우드 (WordCloud)
단어의 빈도수에 따라 글자 크기를 다르게 하여 시각화하는 기법입니다. 텍스트 분석 결과의 "썸네일" 역할을 합니다.

## 11. 코사인 유사도 (Cosine Similarity)
두 문서(벡터)가 얼마나 비슷한지 측정합니다. 벡터 사이의 각도(코사인 값)를 봅니다. 길이가 달라도 방향이 비슷하면 유사하다고 봅니다. (유클리드 거리보다 텍스트에 적합)

## 12. 감성 분석 (Sentiment Analysis)
문장이 긍정적인지 부정적인지 판별합니다. 리뷰 분석, 여론 조사에 쓰입니다. 사전을 이용한 방식과 머신러닝을 이용한 방식이 있습니다.

## 13. N-gram
단어 하나만 보는 게 아니라, 두 개씩 묶어서("Not happy") 봅니다. 문맥 파악에 도움이 됩니다. (Bi-gram, Tri-gram).

## 14. 실습: 영화 리뷰 긍/부정 분류
IMDb 데이터셋 등을 이용하여 리뷰 텍스트를 입력받아 BoW/TF-IDF로 변환 후 로지스틱 회귀로 분류해 봅니다.

## 15. 한국어 처리의 어려움 (KoNLPy)
한국어는 교착어라 조사와 어미가 발달해 있습니다. 자바(Java) 기반의 KoNLPy 설치가 까다로울 수 있지만 필수적입니다.

## 16. 워드 임베딩 (Word Embedding) 맛보기 (Word2Vec)
Count/TF-IDF는 단어의 **의미**를 모릅니다. "왕"과 "남자", "여왕"과 "여자"의 관계를 벡터 연산(왕 - 남자 + 여자 = 여왕)으로 보존하는 딥러닝 기법입니다. (심화 과정)

## 17. 나이브 베이즈 (Naive Bayes) 분류기
텍스트 분류(스팸 필터)에서 전통적으로 강력한 확률 기반 모델입니다. 단어들끼리 서로 독립이라고 가정합니다.

## 18. Python 문법 집중: 리스트 컴프리헨션
`[word for word in text if word not in stopwords]`
텍스트 전처리 코드를 깔끔하게 한 줄로 짜는 법을 익힙니다.

## 19. 토픽 모델링 (LDA) 맛보기
문서 집합 속에 숨어 있는 추상적인 '주제'를 찾아줍니다. 군집화의 텍스트 버전입니다.
