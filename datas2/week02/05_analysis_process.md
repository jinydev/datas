# 05. 분석 파이프라인 (함수와 모듈화)

## 1. 분석은 코딩이 아니다
데이터 분석은 문제 해결을 위한 사고 과정입니다. 코딩은 단지 그 생각을 구현하는 도구일 뿐입니다.

## 2. 1단계: 문제 정의 (Definition)
"매출이 왜 떨어졌을까?" 또는 "A열의 평균은 얼마인가?"와 같이 명확한 질문에서 출발해야 합니다.

## 3. 2단계: 데이터 수집 (Acquisition)
센서, 로그, 사용자 입력 등 다양한 소스로부터 데이터를 가져옵니다. (CSV, JSON, DB 등)

## 4. 3단계: 전처리 (Preprocessing)
"Garbage In, Garbage Out". 쓰레기 데이터가 들어가면 쓰레기 결과가 나옵니다.
결측치(NaN), 오류, 중복을 제거하는 과정입니다.

## 5. 4단계: 탐색적 데이터 분석 (EDA)
모델링 전에 데이터를 눈으로 직접 확인하는 과정입니다. 히스토그램, 산점도 그리기 등이 여기에 해당합니다.

## 6. 5단계: 특성 공학 (Feature Engineering)
"키"와 "몸무게" 데이터를 이용해 "BMI"라는 새로운 의미 있는 지표를 만들어내는 과정입니다.

## 7. 6단계: 분석 및 모델링 (Modeling)
추세를 발견하거나(회귀), 그룹을 나누거나(분류), 미래를 예측하는 단계입니다.

## 8. 7단계: 시각화 및 보고 (Start Reporting)
분석 결과를 이해관계자(상사, 고객)에게 전달하는 단계입니다. 가장 중요한 단계이기도 합니다.

## 9. Python 문법 집중: 함수 (Functions)
이 복잡한 단계들을 정리정돈하기 위해 **함수**를 사용합니다.
`def preprocessing(data):`

## 10. 함수 정의하기 (`def`)
`def` 키워드, 함수 이름, 매개변수, 그리고 콜론(`:`)으로 정의합니다.

## 11. 반환값 (`return`)
함수는 입력(Input)을 받아 처리 후 반드시 결과(Output)를 내뱉어야 합니다.

## 12. 매개변수 (Parameters)
함수가 작동하기 위해 외부에서 받아오는 재료입니다.

## 13. 스코프 (Scope)
함수 내부에서 만든 변수는 함수가 끝나면 사라집니다(Local Variable). 이는 프로그램 전체가 꼬이는 것을 방지합니다.

## 14. 독스트링 (Docstrings)
`""" 이 함수는 데이터를 청소합니다 """`.
좋은 엔지니어는 자신의 코드에 설명서를 달아둡니다.

## 15. 모듈화 (Modular Programming)
긴 코드를 하나의 파일에 다 쓰지 않고, 기능별로 쪼개서 관리하는 것입니다.

## 16. 함수 호출 (Call)
`result = my_function(input)` 형태로 사용합니다.

## 17. 기본 인자 (Default Arguments)
`def power(num, exponent=2):`. 지수를 입력하지 않으면 자동으로 제곱을 구하도록 설정할 수 있습니다.

## 18. 응용 예제: 파이프라인 시뮬레이션
우리는 3개의 함수(`get_data`, `clean_data`, `plot_data`)를 만들어 미니 분석 파이프라인을 구축해 볼 것입니다.

## 19. 도메인 지식의 중요성
의료 데이터를 분석하려면 의학 지식이, 금융 데이터를 분석하려면 경제 지식이 필요합니다. 코딩만큼이나 중요합니다.

## 20. 결론
우리는 코드를 논리적인 사고의 흐름(Pipeline)에 맞춰 구조화하는 법을 배웠습니다. 이제 더 복잡한 문제도 해결할 준비가 되었습니다.
