# 01. 데이터의 본질 (빅데이터 이론과 자료형)

## 1. 빅데이터의 정의
빅데이터는 단순히 '많은 데이터'를 의미하지 않습니다. 이는 기존의 방법으로는 처리, 저장, 분석하기 어려운 거대하고 복잡한 데이터 집합을 의미하는 패러다임의 변화를 뜻합니다.

## 2. 3 V's (빅데이터의 3요소)
전통적인 정의는 다음 세 가지 차원을 포함합니다.
*   **Volume (규모)**: 데이터의 양 (TB, PB 단위).
*   **Velocity (속도)**: 데이터 생성 및 처리 속도 (실시간 스트리밍).
*   **Variety (다양성)**: 정형, 비정형 데이터의 혼재.

## 3. 현대적 정의의 확장
최근에는 **Veracity (정확성)**와 **Value (가치)**를 더해 5V라고 부르기도 합니다.

## 4. 데이터 마이닝 (Data Mining)
데이터 마이닝은 흙(Raw Data) 속에서 금(Information)을 캐내는 과정입니다. 패턴을 발견하고 미래를 예측하는 기술입니다.

## 5. 정형 데이터 vs 비정형 데이터
*   **정형 (Structured)**: 엑셀, RDB처럼 행과 열이 명확한 데이터.
*   **비정형 (Unstructured)**: 텍스트, 이미지, 영상 등 구조가 없는 데이터.

## 6. 데이터 분석 파이프라인
1.  수집 (Acquisition)
2.  전처리 (Preprocessing/Cleaning)
3.  분석/모델링 (Analysis)
4.  시각화/보고 (Visualization)

## 7. Python의 역할 (Glue Language)
Python은 C/C++로 작성된 고성능 엔진과 사용자를 연결해주는 '접착제 언어'입니다. 쉽고 강력하기 때문에 데이터 과학의 표준이 되었습니다.

## 8. Python 문법 집중: 변수는 라벨이다
데이터 분석에서 변수는 값을 저장하는 상자가 아니라, 메모리 어딘가에 있는 객체를 가리키는 **라벨(포스트잇)**입니다.

## 9. 동적 타이핑 (Dynamic Typing)
C나 Java와 달리, Python은 변수의 타입을 미리 선언할 필요가 없습니다.
`x = 10` (자동으로 정수형으로 인식).

## 10. 메모리 관리
Python은 가비지 컬렉션(Garbage Collection) 기능을 통해 사용하지 않는 메모리를 자동으로 정리합니다.

## 11. 정수형 (Integer)
소수점이 없는 숫자입니다. `count = 100`.

## 12. 실수형 (Floating Point)
소수점이 있는 숫자입니다. `temp = 36.5`.
컴퓨터는 이를 부동소수점 방식으로 근사하여 저장합니다.

## 13. 문자열 (String)
글자들의 나열입니다. `name = "Alice"`.
텍스트 마이닝의 대상이 됩니다.

## 14. 불리언 (Boolean)
`True` 또는 `False`. 논리적 판단의 결과입니다.

## 15. 타입 확인 (`type`)
데이터를 다룰 때 가장 먼저 해야 할 일은 타입을 확인하는 것입니다.
```python
x = 10
print(type(x))
```

## 16. 형 변환 (Type Casting)
파일에서 읽어온 숫자는 종종 문자열로 인식됩니다. 이를 숫자로 바꿔주는 과정입니다.
`int("123")` $\rightarrow$ `123`.

## 17. 암시적 변환의 위험성
`print("10" + "10")`은 `20`이 아니라 `"1010"`이 됩니다. 이는 데이터 수집 과정에서 가장 빈번하게 발생하는 오류입니다.

## 18. 응용 예제: 데이터 크기 추산
100만 건의 데이터가 있고 각 행이 100바이트라면, 필요한 메모리는 얼마일까요?

## 19. Python을 이용한 계산
`1_000_000 * 100` 바이트. (숫자 중간의 `_`는 무시됩니다. 가독성을 위함입니다)

## 20. 결론
데이터의 물리적/논리적 특성을 이해하는 것은 대규모 분석의 기초입니다. Python의 자료형 시스템은 이를 유연하게 다룰 수 있게 해줍니다.
