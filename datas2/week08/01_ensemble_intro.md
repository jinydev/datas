# 01. 집단 지성의 힘 (앙상블 학습)

## 1. 앙상블(Ensemble)이란?
여러 개의 머신러닝 모델을 연결하여 하나의 강력한 모델을 만드는 기법입니다.
"백지장도 맞들면 낫다"는 속담처럼, 약한 모델(Weak Learner) 여러 개가 모여 강력한 성능을 냅니다.

## 2. 투표 기반 분류 (Voting Classifier)
*   **하드 보팅(Hard Voting)**: 다수결 원칙. 모델 3개가 각각 [1, 1, 0]을 예측하면 1로 결정.
*   **소프트 보팅(Soft Voting)**: 확률의 평균. [0.9, 0.8, 0.4]라면 평균 0.7로 1이라 결정. 정보량이 더 많아 성능이 더 좋습니다.

## 3. 배깅 (Bagging - Bootstrap Aggregating)
데이터를 랜덤하게 중복 추출(Bootstrap)하여 여러 개의 데이터셋을 만들고, 각각의 데이터셋으로 모델을 학습시킨 뒤 합치는(Aggregating) 방식입니다.
대표 주자: **랜덤 포레스트(Random Forest)**.

## 4. 랜덤 포레스트 (Random Forest)
결정 트리의 숲입니다.
1.  데이터를 샘플링할 때 랜덤성을 줍니다.
2.  노드를 분할할 때 모든 특성을 보지 않고, 일부 특성(Square Root 개수 등)만 무작위로 골라서 최적의 질문을 찾습니다.
이 두 가지 무작위성이 트리의 과대적합을 막고 일반화 성능을 극대화합니다.

## 5. 부스팅 (Boosting)
배깅이 "독립적인" 모델들을 병렬로 학습시킨다면, 부스팅은 "순차적인" 학습입니다.
첫 번째 모델이 틀린 문제(오답)에 가중치를 주어, 두 번째 모델은 그 오답을 맞추는 데 집중합니다. 지속적으로 오류를 개선해 나갑니다.

## 6. 에이다 부스트 (AdaBoost)
초기 부스팅 알고리즘입니다. 오분류된 데이터의 가중치를 높이는 방식을 사용합니다.

## 7. 그래디언트 부스팅 (Gradient Boosting - GBM)
가중치 대신, 이전 모델의 **잔차(Residual, 오차)**를 다음 모델이 예측하도록 학습합니다. 경사 하강법의 원리를 이용합니다. 성능은 강력하지만 속도가 느리고 과대적합 위험이 있습니다.

## 8. XGBoost (Extreme Gradient Boosting)
GBM의 단점(느린 속도)을 병렬 처리와 하드웨어 최적화로 획기적으로 개선한 라이브러리입니다. 캐글(Kaggle) 우승자들이 가장 사랑하는 알고리즘 중 하나입니다.

## 9. LightGBM
Microsoft에서 만들었습니다. XGBoost보다 더 가볍고 빠르면서 성능은 비슷합니다. 대용량 데이터 처리에 적합합니다. 리프 중심 분할(Leaf-wise) 방식을 써서 비대칭 트리를 만듭니다.

## 10. 스태킹 (Stacking)
여러 모델의 예측 결과(Output)를 다시 학습 데이터로 사용하여 최종 메타 모델(Meta Model)이 다시 예측하는 기법입니다. 고수들의 마지막 1% 성능 쥐어짜기 비법입니다.

## 11. 앙상블의 장점
*   단일 모델보다 성능이 뛰어납니다.
*   과대적합에 강합니다 (특히 배깅).
*   안정적입니다.

## 12. 앙상블의 단점
*   모델이 무겁고 학습/예측 시간이 오래 걸립니다.
*   해석이 어렵습니다 (트리가 수백 개라 시각화 불가능). 피처 중요도로만 파악 가능합니다.

## 13. OOB (Out-of-Bag) 점수
랜덤 포레스트에서 부트스트랩 샘플링에 포함되지 않은 나머지 데이터(약 37%)를 이용해 자체 평가하는 점수입니다. 검증 세트가 따로 없어도 성능 측정이 가능합니다.

## 14. 하이퍼파라미터 튜닝의 중요성
앙상블 모델은 튜닝할 나사가 많습니다. `n_estimators`(나무 개수), `learning_rate`(학습률), `max_depth` 등이 성능을 좌우합니다.

## 15. 실습: 와인 품질 분류 (Red vs White)
화학적 특성(도수, 당도, 산도)을 이용해 레드 와인인지 화이트 와인인지 분류하는 문제를 풉니다. 

## 16. 실습: 로지스틱 회귀 vs 랜덤 포레스트
두 모델의 결정 경계와 정확도를 비교해 봅니다. 복잡한 데이터일수록 앙상블이 압도합니다.

## 17. 조기 종료 (Early Stopping)
부스팅 모델 학습 시, 성능이 더 이상 좋아지지 않으면 1000번 반복을 다 채우지 않고 중간에 멈추는 기능입니다. 시간 낭비와 과대적합을 막습니다.

## 18. 불균형 데이터 처리
암 환자 데이터처럼 정답 비율이 9:1로 깨져 있을 때, 앙상블 모델의 `class_weight='balanced'` 옵션이 도움이 됩니다.
