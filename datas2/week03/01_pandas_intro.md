# 01. 엑셀을 넘어서 (Pandas의 세계)

## 1. 엑셀(Excel)의 한계
엑셀은 훌륭한 도구이지만, 100만 행이 넘어가면 느려지고, 자동화하기 어렵습니다. 우리는 프로그래밍 가능한 엑셀이 필요합니다.

## 2. Pandas란?
Python Data Analysis Library의 약자입니다. 금융 데이터 계량 분석을 위해 만들어졌으며, 현재는 모든 데이터 분석의 산업 표준 라이브러리입니다.

## 3. 핵심 자료구조 1: DataFrame (데이터프레임)
엑셀의 '시트(Sheet)'와 똑같습니다. 행(Row)과 열(Column)이 있는 2차원 테이블입니다.

## 4. 핵심 자료구조 2: Series (시리즈)
데이터프레임의 '열(Column)' 하나를 떼어내면 그것이 바로 시리즈입니다. 1차원 배열과 같습니다.

## 5. Pandas 임포트 관례
```python
import pandas as pd
```
모든 데이터 과학자가 이 별칭을 사용하므로, 반드시 따르는 것이 좋습니다.

## 6. 데이터프레임 만들기 (Dictionary 방식)
가장 직관적인 방법은 키(Key)가 열 이름이고, 값(Value)이 리스트인 딕셔너리를 사용하는 것입니다.
```python
data = {'Name': ['Kim', 'Lee'], 'Age': [25, 30]}
df = pd.DataFrame(data)
```

## 7. DataFrame의 해부학: Index (인덱스)
맨 왼쪽에 있는 굵은 숫자들(0, 1, 2...)입니다. 행을 식별하는 주소입니다. 엑셀의 행 번호와 비슷하지만, 문자로도 설정 가능합니다.

## 8. DataFrame의 해부학: Columns (컬럼)
맨 위에 있는 헤더들입니다. 각 열의 이름(변수명)을 나타냅니다.

## 9. 데이터 확인하기: `head()`
데이터가 100만 개일 때 `print(df)`를 하면 화면이 터집니다. `df.head()`는 기본적으로 상위 5개만 보여줍니다.

## 10. 꼬리 확인하기: `tail()`
데이터의 끝부분이 잘 들어왔는지 확인하려면 하위 5개를 보여주는 `tail()`을 씁니다.

## 11. 요약 정보 확인: `info()`
데이터 분석가의 가장 친한 친구입니다.
*   전체 행 개수
*   컬럼별 데이터 타입 (Dtype)
*   결측치 존재 여부 (Non-Null Count)

## 12. 통계적 요약: `describe()`
숫자형 데이터에 대해 기계적인 통계를 한 번에 내줍니다.
평균(mean), 표준편차(std), 최소(min), 최대(max), 사분위수(25%, 50%, 75%).

## 13. 열 선택 (대괄호 `[]`)
단일 열 선택: `df['Name']`. 결과는 Series입니다.

## 14. 점 표기법 (Dot Notation)
`df.Name`으로도 선택 가능하지만, 열 이름에 공백이 있거나 파이썬 예약어와 겹치면 사용할 수 없습니다. 대괄호 표기법을 추천합니다.

## 15. CSV 파일 읽기 (`read_csv`)
실무 데이터는 대부분 파일로 존재합니다. `pd.read_csv('file.csv')` 한 줄이면 엑셀 파일을 파이썬으로 가져올 수 있습니다.

## 16. 인코딩 문제 (Encoding)
한글 파일이 깨진다면? `encoding='cp949'` 또는 `encoding='utf-8-sig'` 옵션을 추가해야 합니다.

## 17. 엑셀 파일 읽기 (`read_excel`)
`.xlsx` 파일도 읽을 수 있습니다. 단, `openpyxl` 라이브러리가 설치되어 있어야 합니다.

## 18. 데이터프레임 저장하기 (`to_csv`)
분석한 결과를 다시 파일로 내보냅니다. `df.to_csv('result.csv', index=False)`.
`index=False`는 불필요한 행 번호 저장을 막아줍니다.

## 19. Python 문법 집중: 객체 지향 (Object-Oriented)
DataFrame은 '객체'입니다. 데이터뿐만 아니라 그 데이터를 다루는 함수(메서드)들(`head`, `info`, `describe`)을 내장하고 있습니다.

## 20. 결론
Pandas는 데이터 분석의 쌀입니다. 앞으로의 모든 과정(전처리, 시각화, 머신러닝)은 Pandas 위에서 이루어집니다.
