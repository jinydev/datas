# 01. 뇌를 흉내낸 기계 (딥러닝 기초)

## 1. 인공지능 vs 머신러닝 vs 딥러닝
*   **AI**: 인간의 지능을 모방하는 모든 기술.
*   **ML**: 데이터를 통해 학습하는 AI의 하위 분야.
*   **DL(Deep Learning)**: 인간의 뉴런 구조를 본뜬 인공신경망(ANN)을 사용하는 ML의 하위 분야.

## 2. 퍼셉트론 (Perceptron)
인공 신경망의 가장 기본 단위입니다. 입력값($x$)에 가중치($w$)를 곱하고 더한 뒤($\sum wx + b$), 활성화 함수를 거쳐 출력($y$)합니다.
초기 퍼셉트론은 XOR 문제(비선형 문제)를 풀지 못해 암흑기를 맞았습니다.

## 3. 다층 퍼셉트론 (MLP - Multi-Layer Perceptron)
퍼셉트론을 여러 층(Layer)으로 쌓으면 XOR 문제도 풀 수 있다는 것이 증명되었습니다.
*   **입력층(Input Layer)**
*   **은닉층(Hidden Layer)**: 특징을 추출하는 층. 딥러닝의 'Deep'은 이 은닉층이 깊다는 뜻입니다.
*   **출력층(Output Layer)**

## 4. 활성화 함수 (Activation Function)
뉴런이 신호를 보낼지 말지 결정하는 문지기입니다. 비선형성을 부여하는 핵심 역할을 합니다.
*   **Sigmoid**: 0~1 사이 값 출력. 기울기 소실 문제로 요즘은 잘 안 씁니다.
*   **ReLU**: 0보다 작으면 0, 크면 그대로 출력. 학습이 빠르고 기울기 소실이 적어 가장 많이 씁니다.
*   **Softmax**: 다중 분류의 출력층에서 사용. 확률의 총합을 1로 만들어줍니다.

## 5. 손실 함수 (Loss Function)
정답과 예측값의 차이를 계산합니다.
*   **MSE**: 회귀 문제용.
*   **Binary Crossentropy**: 이진 분류용.
*   **Categorical Crossentropy**: 다중 분류용.

## 6. 역전파 (Backpropagation)
딥러닝 학습의 핵심 알고리즘입니다. 출력층에서 발생한 오차(Loss)를 거꾸로 전파하여, 각 뉴런의 가중치($w$)를 얼마나 수정해야 할지(미분값) 계산합니다. 체인 룰(Chain Rule)이 적용됩니다.

## 7. 최적화 함수 (Optimizer)
오차를 줄이는 방향으로 가중치를 업데이트하는 방법입니다.
*   **SGD (확률적 경사 하강법)**: 기본.
*   **Adam**: 방향(Momentum)과 보폭(Adaptive LR)을 모두 고려한 만능 해결사. "모르면 일단 Adam을 써라"는 말이 있습니다.

## 8. 텐서플로우 (TensorFlow)와 케라스 (Keras)
Google이 만든 딥러닝 프레임워크입니다.
*   **TensorFlow**: 엔진(Low-level).
*   **Keras**: 운전대(High-level API). 우리는 쉽고 직관적인 케라스를 주로 사용합니다.

## 9. 텐서(Tensor)란?
데이터를 담는 다차원 배열(Numpy Array와 유사)입니다. GPU에서 고속 연산이 가능하도록 최적화되어 있습니다.
0차원(스칼라), 1차원(벡터), 2차원(행렬), 3차원 이상(텐서).

## 10. 에포크(Epoch)와 배치(Batch)
*   **Epoch**: 전체 데이터셋을 한 번 다 공부하는 것.
*   **Batch Size**: 한 번에 몇 문제씩 풀고 채점할 것인가? (보통 32, 64).
*   **Iteration**: 1 에포크를 돌기 위해 필요한 배치의 개수.

## 11. 실습: MNIST 손글씨 분류
딥러닝의 "Hello World". 0~9까지 쓴 손글씨 이미지를 분류하는 모델을 만듭니다. CNN 없이 기본 신경망(Dense Layer)만으로도 98% 정확도가 나옵니다.

## 12. 모델 구조 설계 (`Sequential`)
`model = Sequential()`
`model.add(Dense(128, activation='relu'))`
레고 블록 쌓듯이 층을 쌓습니다.

## 13. 모델 컴파일 (`compile`)
"이 모델은 Adam 최적화 도구를 쓰고, 오차는 Crossentropy로 계산하고, 평가는 정확도(Accuracy)로 하겠다"고 선언하는 단계입니다.

## 14. 모델 학습 (`fit`)
`history = model.fit(X_train, y_train, epochs=10)`
학습 과정의 로그가 `history` 변수에 저장됩니다.

## 15. 학습 곡선 시각화
`history`에 저장된 Loss와 Accuracy 그래프를 그려봅니다. Loss는 떨어지고 Accuracy는 올라가야 정상입니다.

## 16. 과대적합 방지: 드롭아웃 (Dropout)
학습할 때 일부 뉴런을 랜덤하게 꺼버립니다(0으로 만듭니다). 특정 뉴런에 의존하는 것을 막아 과대적합을 강력하게 예방합니다.

## 17. 실무적 조언: GPU의 필요성
행렬 연산이 수백만 번 일어나므로 CPU로는 느립니다. Colab 같은 환경에서 GPU 가속을 켜야 합니다.

## 18. Python 문법 집중: `summary()`
모델의 층 구성과 파라미터 개수를 한눈에 보여줍니다. 내가 만든 모델이 맞는지 확인하는 습관을 들여야 합니다.

## 19. 패션 MNIST (Fashion MNIST)
숫자 MNIST보다 조금 더 어려운 옷, 신발 이미지 분류 데이터셋입니다.

## 20. 결론
딥러닝은 데이터가 많을수록 성능이 계속 좋아지는 확장성(Scalability)이 있습니다. 이제 정형 데이터를 넘어 이미지, 텍스트, 음성 등 비정형 데이터의 시대로 진입합니다.
