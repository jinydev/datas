# 02. Lab: 패션 아이템 분류 (Fashion MNIST)

## 1. 실험실 목표
숫자(MNIST)보다 조금 더 복잡한 옷과 신발 이미지 10종을 분류합니다. 과대적합을 해결하고 성능을 높이는 다양한 기법을 적용해 봅니다.

## 2. 데이터셋 소개 (Fashion MNIST)
28x28 흑백 이미지, 10개 클래스 (T-shirt, Trouser, Pullover, Dress, Coat, Sandal, Shirt, Sneaker, Bag, Ankle boot).
숫자보다 구분하기가 더 까다롭습니다.

## 3. 데이터 로드 및 시각화
클래스 이름(`class_names`)을 리스트로 정의해두고, 이미지를 출력할 때 매핑해서 보여줍니다.
데이터의 형태(Shape)와 픽셀 범위(0~255)를 확인합니다.

## 4. 데이터 전처리
*   픽셀값 정규화 (/ 255.0)
*   검증 데이터 분리 (Train에서 20% 떼어내기)

## 5. 베이스라인 모델 (Basic MLP)
`Flatten` -> `Dense(128)` -> `Dense(10)` 구조로 가장 단순한 모델을 만들고 성능을 측정합니다. (약 85~87% 예상)

## 6. 콜백 (Callback) 함수 활용
*   `ModelCheckpoint`: 학습 중 성능이 가장 좋은(Val Loss가 가장 낮은) 모델을 자동으로 저장합니다.
*   `EarlyStopping`: 성능이 더 이상 좋아지지 않으면(patience=3) 학습을 조기에 멈춥니다.
이 두 가지는 실무 필수입니다.

## 7. 심층 신경망 (Deep NN)
은닉층을 더 깊게 쌓아봅니다. (128 -> 64 -> 32). 모델이 깊어지면 더 복잡한 특징을 학습할 수 있습니다.

## 8. 오차 분석 (Confusion Matrix)
어떤 옷을 가장 많이 헷갈려 할까요?
보통 'Shirt'와 'T-shirt', 'Pullover'와 'Coat'를 혼동합니다. 혼동 행렬 히트맵을 그려 확인합니다.

## 9. 잘못 예측한 이미지 시각화
모델이 "스니커즈"라고 우겼는데 실제로는 "샌들"인 이미지를 직접 눈으로 봅니다. (실제로 애매하게 생긴 경우가 많습니다.)

## 10. 함수형 API (Functional API)
`Sequential`은 일직선 구조만 가능하지만, `Model(inputs=..., outputs=...)` 방식의 함수형 API는 층을 건너뛰거나(Skip Connection) 입력을 여러 개 받는 복잡한 모델을 만들 수 있습니다. 맛보기로 소개합니다.

## 11. 하이퍼파라미터 튜닝 (Keras Tuner 소규모)
은닉층의 노드 수를 32로 할지 256으로 할지 고민될 때, 코드로 여러 조합을 실험해 봅니다.

## 12. 배치 정규화 (Batch Normalization)
각 층을 통과할 때마다 데이터 분포가 쏠리는 현상을 막기 위해, 층 중간중간에 강제로 정규화를 해주는 층입니다. 학습 속도와 성능을 비약적으로 높여줍니다.
`Dense` -> `BatchNormalization` -> `Activation` 순서로 씁니다.

## 13. 옵티마이저 비교 실험
`Adam` vs `SGD` vs `RMSprop`.
같은 모델이라도 최적화 도구에 따라 학습 속도와 최종 성능이 다름을 그래프로 비교합니다.

## 14. 모델 저장과 복원 (`save`, `load_model`)
열심히 학습시킨 모델을 `my_fashion_model.h5` 파일로 저장하고, 나중에 불러와서 다시 예측해 봅니다.

## 15. 예측 확률 분포 확인
Softmax의 출력값은 10개 클래스에 대한 확률입니다. `[0.01, ..., 0.9, ...]` 그래프를 그려 모델의 확신 정도를 봅니다.

## 16. 실습: 나만의 손글씨/옷 이미지 테스트
그림판으로 그린 옷이나 손글씨를 모델에 넣어봅니다. (이미지 크기를 28x28로 리사이징해야 함을 주의)

## 17. CNN 예고
MLP는 이미지를 1줄로 펴서(Flatten) 처리하기 때문에 위치 정보(팔이 몸통 옆에 있다 등)가 사라집니다. 다음 시간에 배울 CNN이 이 문제를 해결해 줄 것입니다.

## 18. Python 문법 집중: `np.argmax`
확률 배열 `[0.1, 0.8, 0.1]`에서 가장 큰 값의 인덱스(1)를 뽑아내는 필수 함수입니다.

## 19. 과제
정확도 90% 돌파하기. 층을 늘리거나, 뉴런 수를 조절하거나, Dropout 비율을 바꿔보세요.

## 20. 결론
신경망은 레고 블록과 같습니다. 층(Layer), 활성화 함수, 옵티마이저라는 블록을 어떻게 조립하느냐에 따라 무한한 가능성이 열립니다.
