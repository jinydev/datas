# 03. 추상화와 변수 (확장 가능한 도형)

## 1. 하드코딩(Hard-Coding)의 한계
지난 시간에 우리는 `[1, 2]`처럼 숫자를 직접 입력했습니다. 이를 **하드코딩**이라고 합니다. 이는 수정하기 어렵고 유연하지 못합니다.

## 2. 변수(Variable)의 도입
변수는 데이터를 담는 그릇입니다. 인간이 이해하기 쉬운 이름을 메모리 상의 값과 연결해 줍니다.

## 3. Python 문법 집중: 할당(Assignment) `=`
수학에서 `=`는 '같다'를 의미하지만, 프로그래밍에서는 **할당**을 의미합니다.
`size = 10`은 "size라는 이름의 상자에 10을 넣어라"라는 뜻입니다.

## 4. 그래픽에 변수 적용하기
`[1, 2]` 대신 `[0, size]`를 시각화한다고 생각해 봅시다. 이제 `size = 20`으로 값을 바꾸면 그래프의 크기도 자동으로 2배가 됩니다.

## 5. 동적 좌표 계산 (Dynamic Coordinates)
변수와 산술 연산을 결합해 봅니다.
`x = [0, size, size, 0, 0]`

## 6. Python 문법 집중: 산술 연산자
Python은 훌륭한 계산기입니다.
`+` (더하기), `-` (빼기), `*` (곱하기), `/` (나누기).

## 7. 원점(Origin)의 개념
컴퓨터 그래픽에서 위치는 보통 원점 $(0, 0)$을 기준으로 상대적으로 정의됩니다.

## 8. 이동(Translation)
도형을 이동시키려면 모든 좌표에 특정한 값(Offset)을 더하면 됩니다.
$x_{new} = x_{old} + x_{offset}$

## 9. Python 문법 집중: 변수 업데이트
`pos = 5`
`pos = pos + 1` (이제 pos는 6이 됩니다).

## 10. 문자열 변수
변수에는 숫자뿐만 아니라 글자도 담을 수 있습니다.
`title = "Variable Graph"`
`plt.title(title)`

## 11. 정수(Integer) vs 실수(Float)
*   `10`은 **정수 (Integer)**: 셀 수 있는 수.
*   `10.5`는 **실수 (Float)**: 연속적인 수.

## 12. 타입 확인 (`type`)
`type(variable)` 함수를 사용하면 변수의 자료형을 알 수 있습니다.

## 13. 네이밍 컨벤션 (Snake Case)
Python에서는 변수 이름을 지을 때 소문자와 밑줄(`_`)을 사용하는 `snake_case`를 선호합니다.
Good: `my_variable`, `box_size`
Bad: `myVariable`, `BoxSize`

## 14. 스케일 팩터 (Scale Factor)
배율을 조절하는 변수 `scale`을 도입해 봅시다.
`coord * scale`

## 15. `input()` 함수
사용자로부터 직접 데이터를 입력받을 수도 있습니다.
`val = input("크기를 입력하세요: ")`

## 16. 형 변환 (Type Casting)
`input()`으로 받은 데이터는 무조건 문자열입니다. 이를 숫자로 바꿔줘야 계산할 수 있습니다.
`val = int(val)`

## 17. 인터랙티브 플롯팅
사용자의 입력에 따라 모양이 바뀌는 그래프를 만들 수 있습니다.

## 18. 디버깅과 출력
변수 값이 의심스러울 때는 `print(variable)`로 내부를 들여다보는 것이 가장 좋은 청진기입니다.

## 19. 응용 예제: 파라메트릭 사각형
사용자가 `size`와 시작 `position`을 정의하면 그에 맞춰 그려지는 사각형을 만들어 봅시다.

## 20. 결론
변수를 사용함으로써 우리는 로직을 한 번만 작성하고 무한히 재사용할 수 있게 되었습니다. 이것이 프로그래밍의 핵심입니다.
