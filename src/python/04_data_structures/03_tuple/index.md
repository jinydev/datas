# 3.4.3 튜플

## 학습목표
본 장에서는 일단 틀이 만들어진 후 내용물이 절대 외부 조작에 의해 긁히거나 변형, 수정되지 않는(Immutable) **'튜플(Tuple)'**의 보안 및 태생적 안전성의 가치를 명확히 파악합니다. 더 나아가 이 굳건한 성질을 이용해 데이터를 간편히 가방에 싸고 또 한 번에 푸는 마법 같은 **패킹(Packing)과 언패킹(Unpacking)** 문법의 짜릿함을 몸소 체험하게 됩니다.

## 튜플 개요와 활용

파이썬의 튜플(tuple)은 리스트와 유사하지만, 한 번 생성되면 변경할 수 없는(immutable) 시퀀스 자료형이다. 튜플은 괄호 `( )`로 표현되며, 각 요소는 쉼표 `,`로 구분된다. 튜플은 다음과 같은 특징을 갖는다.

- **불변성(immutable)**: 한 번 생성된 튜플은 수정할 수 없기 때문에, 데이터 무결성을 보장하고 의도치 않은 변경을 방지
- **성능**: 리스트보다 메모리 사용이 적고, 읽기 속도가 빠름
- **해시 가능성(Hashable)**: 튜플은 딕셔너리의 키나 집합의 원소로 사용될 수 있으며, 이를 통해 유용한 데이터 구조를 만들 수 있음

괄호를 둘러싸며 생략할 수 있다.

```python
tp1 = 1, 2, 3
tp2 = (10, 20, 30)
print(tp1, tp2)
```
**출력:**
```
(1, 2, 3) (10, 20, 30)
```

```python
tp3 = tuple((10, 20, 30))
tp4 = tuple(['py', 'R'])
print(tp3, tp4)
```
**출력:**
```
(10, 20, 30) ('py', 'R')
```

튜플은 불변성을 가지고 있기 때문에, 요소의 추가, 삭제, 수정 등의 연산은 지원되지 않는다. 하지만 튜플 간의 연결이나 반복 등은 가능하다.

```python
tuple1 = (1, 2, 3)
tuple2 = (4, 5, 6)
concatenated_tuple = tuple1 + tuple2
concatenated_tuple
```
**출력:**
```
(1, 2, 3, 4, 5, 6)
```

## 튜플의 불변성 (Immutability)과 활용

'불변성'은 값이 한 번 생성되면 생성되는 순간 그 안의 내용물이 고정되어 버린다는 뜻입니다. 파이썬에는 다른 형태의 상수 선언 문법(C언어의 `const`, Java의 `final`)이 없는 대신, 튜플을 사용하여 **데이터의 묶음(구조물) 자체를 안전하게 보호**할 수 있습니다.

- **안전 장치 기능**: 읽기 전용(Read Only)이 되므로 누군가 실수로 핵심 데이터(예: 서버 설정 정보, 화면의 X/Y 좌표 등)를 수정하거나 삭제하는 것을 시스템 레벨에서 막아줍니다.
- **구조체(Struct) 형태 활용**: 서로 다른 데이터 타입을 단단하게 묶어서 의미 있는 단위로 만듭니다. (예: `('Alice', 25, 165.5)`처럼 이름, 나이, 키를 하나로 취급)
- **딕셔너리 키(Key) 사용 가능**: 리스트는 내용물이 변할 수 있어서 딕셔너리의 열쇠(Key)로 사용할 수 없지만, 튜플은 자물쇠가 걸려있기 때문에 딕셔너리 키로 쓸 수 있습니다. (Hashable 속성)

## 패킹과 언패킹 (Packing & Unpacking)

파이썬에서 가장 우아하고 편리한 문법 중 하나입니다. **패킹**은 여러 물건을 가방 한 개에 넣어서 싸는 것이라면, **언패킹**은 가방에서 물건을 꺼내 각자 변수들에게 나누어 주는 것을 의미합니다.

**1. 패킹 (Packing)**
여러 개의 값을 하나의 튜플로 묶는 것입니다. 위에서 괄호를 생략(`tp1 = 1, 2, 3`)해도 튜플이 생성되었던 것이 바로 패킹 문법입니다.

**2. 언패킹 (Unpacking)**
튜플에 묶인 값을 좌변의 여러 변수에 한 번에 순서대로 할당해 줍니다.
```python
x, y, z = (10, 20, 30)
print(y) # 3.4.3 ```

이 언패킹의 강력함을 이용하면 두 변수의 값을 임시 변수(`temp`) 없이 바로 교환(Swap)할 수 있습니다.
```python
a = 10
b = 20
a, b = b, a # (20, 10)이라는 튜플을 임시로 메모리에 패킹했다가 a, b에 각각 언패킹함
print(a, b) # 3.4.3 10
```

## 메모리 효율성과 튜플의 이점 (Efficiency)

수정 기능들이 제외되어 있기 때문에, 동일한 데이터를 가지고 있더라도 튜플은 리스트보다 메모리를 적게 차지하며 자료 생성 및 처리 속도도 미세하게 더 빠릅니다.

```python
import sys

ls = [1, 2, 3, 4, 5]
tp = (1, 2, 3, 4, 5)

print(sys.getsizeof(ls)) # 리스트가 차지하는 메모리 크기 (더 무겁다)
print(sys.getsizeof(tp)) # 튜플이 차지하는 메모리 크기 (가볍다)
```

수정할 필요가 없는 정적 데이터를 다룰 때는 습관적으로 리스트 `[]` 대신 튜플 `()`을 사용하는 것이 더 파이썬다운 효율적인 코딩 습관(Pythonic)입니다.

## 정리
프로그래밍 초보일 때는 단순하게 값의 추가나 뺄셈조차 마음대로 안 되는 답답하고 낡은 리스트의 하위 호환 구조 정도로 치부하기 쉽습니다. 그러나 이 엄격한 **'수정 불가능성(Immutable)'**이야말로 치명적인 메모리 데이터 변조 해킹이나 의도치 않은 시스템 크래시를 원천 차단하는 가장 믿음직한 방패막 역할을 수행합니다. 게다가 차지하는 메모리 용량마저 몹시 슬림한 이 튜플 메커니즘의 언패킹 미학을 사랑하게 된다면 비로소 진정한 '파이써닉(Pythonic)'한 개발의 미학에 눈을 뜬 것입니다.
