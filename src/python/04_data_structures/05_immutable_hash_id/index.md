# 3.4.5 수정 불가능(immutable)과 함수 hash() id()

## 학습목표
본 장에서는 파이썬 코어의 깊숙한 내부 메모리 지형에서 값이 절대로 변할 수 없는(Immutable) 객체 무리들이 어떠한 수학적 암표 처리 과정을 거쳐 절대 변조되지 않는 보안 식별자(**Hash**)를 보장받는지 그 심연의 근본 원리를 관찰합니다. 특수 내장 함수 `id()`를 직접 호출함으로써, 포장지 껍데기(값 구조)가 아닌 진짜 메모리 속의 유일한 하드웨어적 실체를 눈으로 추적 비교하는 개발자의 아주 깊고 단단한 기본기를 다집니다.

파이썬에서 수정 불가능하다는 immutable(불변)은 변경할 수 없는 객체를 가리킨다. immutable 객체는 생성된 후에 그 값을 변경할 수 없다. 예를 들어, 정수(`int`), 부동소수점(`float`), 문자열(`str`), 튜플(`tuple`) 등은 immutable 객체이다. 이러한 객체는 생성되면 값을 변경할 수 없으며, 변경하려고 하면 새로운 객체가 생성된다.

수정 불가능한 immutable 속성은 객체의 안전성과 불변성을 보장하여 예상치 못한 변경으로부터 객체를 보호한다. 따라서 파이썬에서는 해시 가능한 객체만 딕셔너리의 키나 집합의 원소로 사용할 수 있는데, 이러한 제한은 immutable한 특성을 가진 객체에 의해 부여된다.

- **해시 가능한 객체(hashable objects)**: 딕셔너리(dict)의 키가 가능
- **해시 가능한 객체(hashable objects)**: 집합(set)의 원소로 가능

파이썬 내장 함수 `hash(object)`는 주어진 객체 `object`의 해시 값을 반환한다. 해시 값은 객체의 내용에 기반하여 생성된다. 해시 함수는 동일한 입력에 대해서 항상 동일한 해시 값을 반환한다. 따라서, 동일한 값이면 항상 동일한 해시 값을 가지게 된다. 해시 값의 고유성은 데이터 구조에서 빠른 검색 및 비교를 위한 핵심 요소 중 하나이다.

다음 코드처럼 수정 가능한 객체인 사전이나 리스트, 집합은 `hash()` 값을 가질 수 없다.

```python
print(hash([1, 2, 3]))
```
**오류:**
```
TypeError: unhashable type: 'list'
```

```python
print(hash({'one': 1}))
```
**오류:**
```
TypeError: unhashable type: 'dict'
```

그러나 수정 불가능한 객체인 기본 자료형과 튜플은 고유한 `hash` 값이 있다.

```python
print(hash('py'), hash((1, 2)))
```
**출력 (예시):**
```
-1857589116887210153 -3550055125485641917
```

수정 불가능한 immutable 객체와 함수 `id()`는 파이썬에서 객체의 고유 식별자(unique identifier)를 이해하는 데 중요한 역할을 한다. 먼저, immutable 객체는 변경할 수 없는 객체를 말한다. 이러한 객체는 생성된 후에 그 값을 변경할 수 없으며, 변경하려고 하면 새로운 객체가 생성된다.

함수 `id()`는 파이썬에서 모든 객체의 고유 식별자(unique identifier)를 반환한다. 이 식별자는 객체가 메모리 상에서의 위치를 나타내는 정수 값이다. 따라서, `id()` 함수를 사용하면 두 개의 객체가 동일한 메모리 위치를 가리키는지 여부를 확인할 수 있다.

```python
help(id)
```

수정 불가능한 객체는 한 번 생성되면 그 값을 변경할 수 없으므로, 동일한 값의 immutable 객체는 메모리 상에서 하나의 위치에만 저장될 수 있다. 이는 `id()` 함수를 사용하여 두 개의 immutable 객체의 식별자를 비교할 때 유용할 수 있다. 만약 두 객체의 `id()`가 동일하다면, 두 객체는 동일한 메모리 위치를 가리키고 있음을 의미한다. 따라서 두 객체는 동일한 것이다.

예를 들어, 다음과 같은 코드를 살펴보자. 변수 `x`와 `y`는 모두 정수값 10을 가리킨다. 이 경우, `id(x)`와 `id(y)`는 동일한 값을 반환한다. 이는 파이썬의 메모리 관리 방식으로 인해, 정수값 10은 불변이므로 한 번 생성된 객체는 메모리 상에서 하나의 위치에만 저장되기 때문이다.

```python
x, y = 10, 10
print(id(x), id(y))
```
**출력 (예시):**
```
140704255284296 140704255284296
```

따라서, immutable한 객체와 함수 `id()`를 함께 사용하면 객체의 메모리 위치를 확인하여 두 객체가 동일한 값을 가리키는지 여부를 쉽게 판단할 수 있다.

## 정리
"왜 편리무쌍한 딕셔너리의 Key 자리 스폿이나 깐깐한 중복 방지망인 Set 괄호 속에는 유연한 수정 리스트를 통째로 절대 밀어 넣고 우겨넣어 등록할 수 없을까?"라는 흔하고 숱한 파이썬 초보 입문자들의 거대한 한계 의문은 여기서 깨달은 해시(Hash)의 변형 불가 룰의 쇠고랑을 직시함으로써 완전히 해갈되었습니다. 변수가 가리키고 매달린 메모리 공간의 실제 위치 주소(`id`)를 아는 것은 나중에 머신러닝 데이터 객체 덩어리를 잘라 복사하고 붙이는 정제 과정에서 어처구니 없는 원본 훼손 사고를 막아내는 가장 중대한 기본 소양이 됩니다.
