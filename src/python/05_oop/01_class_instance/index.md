# 3.5.1 클래스와 인스턴스 (Class & Instance)

## 학습목표
본 장에서는 객체를 잉태하는 붕어빵 틀인 **'클래스(Class)'**와 그 틀에서 찍혀 나온 구체적인 실체인 **'인스턴스(Instance)'**의 차이를 명확하게 깨우칩니다. 객체의 초기 상태를 세팅하는 생성자(`__init__`)의 역할과, 모든 객체 내부 메서드 파라미터에 반드시 등장하는 `self` 키워드의 진짜 의미를 가장 투명하게 이해합니다.

파이썬의 세계에서는 **모든 것이 객체(Object)**입니다. 숫자, 문자열, 리스트, 나아가 함수까지도 모두 객체입니다. 이 객체를 만들어내는 공장이자 설계도가 바로 **클래스(Class)**입니다. 이번 단원에서는 클래스와 인스턴스의 개념, 그리고 이들을 구성하는 기본 요소에 대해 알아봅니다.

## 3.5.1 클래스와 인스턴스의 개념

- **클래스 (Class)**: 객체를 만들기 위한 **청사진(Blueprint)**이나 **틀(템플릿)**입니다. 이 안에는 객체가 가질 상태(변수)와 행동(메서드)이 정의되어 있습니다.
- **인스턴스 (Instance)**: 클래스라는 틀을 기반으로 컴퓨터 메모리에 실제로 할당되어 만들어진 **구체적인 실체**를 의미합니다. (즉, 객체 중에서 특정 클래스를 통해 만들어진 객체를 지칭할 때 인스턴스라고 부릅니다.)

예를 들어, `붕어빵 틀`이 클래스라면 이 틀에서 찍어져 나온 팥 붕어빵, 슈크림 붕어빵은 `인스턴스`입니다. 붕어빵 틀은 하나지만 찍어내는 붕어빵은 수만 개가 될 수 있습니다.

## 3.5.1 클래스 정의와 인스턴스 생성

클래스를 정의할 때는 `class` 키워드를 사용하며 파이썬의 클래스 이름은 보통 파스칼 케이스(PascalCase: 첫 글자가 대문자인 방식)를 사용합니다.

```python
class Robot:
    pass

# 3.5.1 인스턴스 생성
robot_a = Robot()
robot_b = Robot()

print(type(robot_a)) # <class '__main__.Robot'>
```

`robot_a`와 `robot_b`는 모두 `Robot` 클래스의 인스턴스지만, 완전히 독립적인 두 개의 다른 객체입니다.

## 3.5.1 생성자 (Constructor): `__init__`

객체가 생성될 때 가장 먼저 자동으로 호출되는 특별한 메서드를 **생성자**라고 부릅니다. 파이썬에서는 `__init__` 이라는 이름을 가집니다. 생성자는 주로 객체의 초기 상태(초기 속성값)를 설정하는 데 쓰입니다.

```python
class Robot:
    def __init__(self, name, color):
        self.name = name
        self.color = color
        print(f"[{self.name}] 로봇이 조립되었습니다! (색상: {self.color})")

robot1 = Robot("R2D2", "White/Blue")
robot2 = Robot("C3PO", "Gold")
```

위의 예시처럼 설계도(클래스)는 하나지만 그 속성(이름, 색상)을 다르게 부여하면 개성 있는 여러 인스턴스를 무한대로 생산할 수 있습니다.

## 3.5.1 `self`의 의미

파이썬 객체지향에서 클래스 내부의 메서드(함수)를 정의할 때 첫 번째 매개변수로 반드시 들어가는 것이 바로 `self`입니다.
- **`self`는 자기 자신(인스턴스 그 자체)을 의미합니다.**
- 위 예제에서 `self.name = name`은 `방금 만들어진 이 인스턴스의 이름(name) 속성에, 매개변수로 들어온 name 값을 대입하라`는 뜻입니다.

## 3.5.1 메서드 (Method)

클래스 안에 정의된 함수를 메서드라고 부릅니다. 인스턴스가 할 수 있는 '행동'을 나타냅니다.

```python
class Robot:
    def __init__(self, name):
        self.name = name
        self.power = 100
        
    def say_hello(self): # 반드시 첫 파라미터는 self 여야 합니다
        print(f"안녕하세요! 저는 {self.name}입니다. 배터리는 {self.power}% 남았습니다.")
        
    def move(self, distance):
        self.power -= distance
        print(f"{distance}만큼 이동했습니다. (잔여 배터리: {self.power}%)")

# 3.5.1 활용
bot = Robot("Wall-E")
bot.say_hello()
bot.move(10)
bot.say_hello()
```

이와 같이 클래스를 활용하면 데이터(속성)와 그것을 통제하는 로직(메서드)을 하나의 깔끔한 덩어리(객체)로 **캡슐화**하여 관리할 수 있습니다. 코드가 복잡해질수록 이러한 분리 방식은 엄청난 유지보수성의 향상을 가져옵니다.

## 정리
우리는 이 장을 통해 파이썬이 세상을 바라보는 방식인 "모든 것은 객체(Object)다"라는 철학을 코드로 직접 증명했습니다. 자동차라는 추상적인 개념(Class) 하나만 잘 설계해 두면, 빨간 마티즈(Instance 1)와 파란 그랜저(Instance 2)를 코드 단 두 줄만으로 무한정 찍어내어 각각 완전히 다른 상태(속성)로 격리 관리할 수 있다는 이 엄청난 생산성을 반드시 기억하시기 바랍니다.
