# 3.1.4 변수 이름 조건

프로그래밍 언어에서 개발자가 직접 만드는 변수, 함수, 클래스 등의 이름을 **식별자(Identifier)**라 합니다. 

파이썬에서 변수 이름을 지을 때는 시스템이 강제하는 **규칙(Rules)**과, 개발자들 사이에서 권장되는 **관례(Conventions)**를 모두 숙지해야 합니다. 이러한 규칙들은 코드의 가독성과 유지보수성을 극적으로 높여줍니다.

## 변수 이름 짖기

### 변수명 필수 규칙

파이썬의 식별자는 다음의 강제 규칙을 따라야 하며, 이를 위반할 경우 `SyntaxError`가 발생합니다.

1. **사용 가능 문자**: 영문 알파벳(대소문자 구별됨), 숫자, 밑줄(`_`)로만 구성해야 합니다. (`%`, `$`, `#` 등의 특수문자와 공백은 절대 사용할 수 없습니다.)
2. **첫 글자 조건**: 첫 글자는 반드시 알파벳이나 밑줄(`_`)로 시작해야 하며, **숫자로는 절대로 시작할 수 없습니다.**
3. **대소문자 구분**: `data`와 `Data`는 서로 완전히 다른 변수로 취급됩니다.
4. **마침표 연산자 금지**: 마침표 `.`는 객체의 속성이나 메서드에 접근하는 특수 연산자이므로 식별자에 포함할 수 없습니다.
5. **예약어 사용 금지**: 파이썬 문법에서 이미 사용 중인 예약어(Keywords: `if`, `for`, `class`, `while`, `True`, `False` 등)는 변수 이름으로 지정할 수 없습니다.

### 실습
실습을 통하여 변수의 규칙을 익혀봅시다.

```python
# 3.1.4 올바른 변수명 작성 예시
age = 19
my_age = 20
_height = 172.3
_26asian = 2026   # 첫 글자가 숫자만 아니면 숫자가 섞여도 무방함

# 3.1.4 잘못된 변수명 작성 예시 (구문 오류 발생)
26worldcup = 2026 # 숫자로 시작하면 에러
my.age = 21       # .은 식별자에 사용할 수 없음
my credits = 18   # 공백 문자 사용 에러
if = 10           # 파이썬 예약어(키워드) 사용 에러
```


## 변수명 작성 스타일과 권장 관례

오류가 나지 않는다고 해서 모두 '좋은 변수명'은 아닙니다. 
파이썬 커뮤니티(주로 PEP 8 스타일 가이드)에서 권장하는 코딩 스타일은 다음과 같습니다.

1. **의미 있는 명명**: 그 용도나 저장된 데이터를 명확히 설명하도록 의미 있는 단어를 사용해야 합니다.
    - 좋은 예시: `user_age`, `total_price`, `is_active`
    - 나쁜 예시: `a`, `b`, `x1`, `temp2`
2. **스네이크 케이스 (snake_case)**: 파이썬에서 일반적인 변수명과 함수명을 지을 때 가장 표준으로 삼는 방법입니다. **모든 글자를 영문 소문자로 작성**하며, 단어와 단어 사이를 **밑줄(`_`)**로 이어붙입니다.
    - 예시: `student_name`, `calculate_sum`
3. **카멜 케이스 (CamelCase / PascalCase)**: 각 단어의 **첫 글자만 대문자**로 시작하고 밑줄 없이 작성하는 방법입니다. 파이썬에서는 주로 **클래스(Class)**의 이름을 지을 때만 제한적으로 사용합니다.
    - 예시: `DataProcessor`, `UserAccount`

이러한 규칙을 엄격하게 따르는 것은 코드를 보다 효율적으로 작성하고, `다른 개발자들과 협업 시 서로 원활하게 소통`하기 위한 기본기능입니다.
> 이를 코딩 컨벤셜 이라고 합니다.
