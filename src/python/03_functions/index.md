# 3.3 함수 (Functions)

![함수 마법 오븐](./img/functions_intro.png)

## 학습목표
내가 짠 파이썬 코드가 수십 줄, 수백 줄 길어지기 시작하면 코드의 중복과 복잡함에 압도당하게 됩니다. 코드를 재사용 가능한 단위로 묶어서 블록화하는 핵심 개념인 **'함수(Function)'**에 대해 학습합니다. 본 장에서는 마법 공간인 함수를 직접 설계하는 방법(`def`)부터, 재료를 넘겨받는 매개변수(Parameter), 그리고 완제품을 뱉어내는 반환값(`return`)의 메커니즘을 완벽히 이해합니다. 또한 변수들의 생존 공간인 스코프(Scope)의 개념과 일회용 가벼운 껍데기인 **람다(Lambda) 함수**까지 폭넓게 배웁니다.

---

## 📚 세부 학습 목차

### [3.3.1 함수란 무엇인가?](./01_function_concepts/)
모든 프로그래밍 언어의 뼈대인 '함수'의 개념을 수학의 일대일 대응 다이어그램이 아닌, 원재료(Input)를 넣으면 마법(Logic)을 부려 완제품(Output)을 뱉어내는 '마법 오븐'의 비유를 통해 능동적이고 직관적으로 이해합니다.

### [3.3.2 함수 선언과 사용](./02_function_usage/)
남이 만든 함수(print, type 등)를 쓰는 소비자 입장에서 벗어나, 파이썬에서 `def` 키워드를 활용해 나만의 커스텀 함수 기계를 직접 설계하는 창조자가 되어봅니다. 매개변수 기본값(Default) 설정과, 여러 개의 결과값을 한 번에 던져주는 파이썬 특유의 다중 반환(Tuple Return) 실전 문법을 익힙니다.

### [3.3.3 변수 스코프와 람다 함수](./03_lambda_functions/)
왜 함수 안에서 만든 변수는 밖에서 쓸 수 없을까요? 함수라는 밀실 안의 '지역 변수'와 바깥세상의 '전역 변수' 등 데이터가 살아 숨 쉬는 공간(Scope)의 범위를 파악합니다. 더불어 데이터 데이터 전처리나 간단한 정렬을 할 때 이름조차 짓기 귀찮은 일회용 함수인 `lambda`의 작고 매운 맛을 몸에 익힙니다.

### [3.3.4 (참고) 내장 함수와 메서드 활용](./04_built_in_functions/)
파이썬 생태계에 입장하자마자 언제든 뽑아 쓸 수 있도록 든든하게 제공되는 핵심 내장 함수(`len()`, `max()`, `sum()`, `enumerate()` 등)들의 활용법을 살펴봅니다. 코드의 길이를 드라마틱하게 줄여줄 치트키 창고입니다.

---

## 🎉 정리
축하합니다! 함수를 직접 만들고 다루는 법을 깨우쳤다는 것은, 드디어 코드를 무작정 일직선으로 짜내려가는 초보 티를 벗어나 '나만의 요리법과 도구 상자'를 패키징할 줄 아는 **설계자(Architect)** 영역에 들어섰다는 뜻입니다. 앞으로 작성될 모든 대규모 AI 코드나 복잡한 알고리즘은 전부 이 함수 단위로 예쁘게 조각나고 조립되어 거대한 시스템을 굴리게 될 것입니다.
