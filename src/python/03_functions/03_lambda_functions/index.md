# 3.3.3 변수 스코프와 람다 함수

## 학습목표
본 장에서는 함수 내부 코드와 바깥세상을 엄격히 가로막는 **'변수의 유효 범위(Scope)'** 개념과 닫힌 문을 부수는 `global` 키워드의 작동 원리를 명확히 파악합니다. 더 나아가, `def`로 이름표조차 붙일 시간 없이 즉석에서 1줄로 생성하고 폐기하는 강력한 일회용 익명 함수, **람다(Lambda)**의 용도를 전격 해부합니다.

함수를 작성하다 보면 데이터의 생존 범위(스코프)와, 아주 짧게 쓰고 버릴 익명 함수가 필요한 순간이 옵니다.

## 변수의 유효 범위 (Scope)와 Global 키워드

일반적으로 함수 안에 선언된 변수들은 바깥의 세상(전역 공간)에 전혀 영향을 주지 않고 철저하게 고립됩니다. 안쪽에 생성된 함수 공간을 **지역 스코프(Local Scope)**라고 하며 바깥쪽 공간을 **전역 스코프(Global Scope)**라고 부릅니다.

```python
y = 2 # 이 y는 바깥세상의 '전역 변수'입니다

def my_func(x):
    y = 10 # 전혀 다른 고립된 '지역 변수'
    return x + y

print(my_func(5)) # 결과: 15
print(y)          # 지역 변수가 아무리 날뛰어도 전역변수는 그대로: 2
```

하지만, 간혹 함수 안의 복잡한 로직을 타고 난 뒤 함수 바깥 스코프에 있는 전역 변수의 상태를 물리적으로 수정해야 하는 상황이 발생합니다. 그럴 때는 함수 내부에서 `global` 키워드를 선언하여 **"지금부터 등장하는 `y`라는 변수는 저 바깥에 있는 진짜 전역 변수 `y`를 조종하겠다"** 라고 명시해야만 합니다.

```python
y = 2 

def update_global():
    global y # 바깥 세상의 y를 가져오겠다!
    y = y * 10
    print("내부에서 수정된 전역 변수:", y) 

update_global()
print("바깥에서도 변경된 전역 변수:", y)

# 내부에서 수정된 전역 변수: 20
# 바깥에서도 변경된 전역 변수: 20
```

## 람다(Lambda) 익명 함수

만약 코드가 단 1줄, 2줄에 그치는 매우 작은 함수 껍데기라면 굳이 복잡하게 `def`로 이름 짓지 않고 `lambda` 익명 함수로 즉석에서 처리해 메모리 등록을 건너뛸 수 있습니다. 

람다 함수는 이름이 없는 익명 함수로, 주로 함수형 프로그래밍에서의 일회성 인자 전달을 위해 사용됩니다.

### 람다 함수 선언 문법

```python
# lambda 매개변수들: 결과값표현식
```

### 활용 예제

```python
import math

# 일반 함수의 경우
def calc_hypotenuse(a, b):
    return math.sqrt(a**2 + b**2)

# 람다 함수로 일회성 처리
result = (lambda a, b: math.sqrt(a**2 + b**2))(3, 4)
print(result) # 5.0
```

람다 함수는 보통 파이썬 내장형 함수(`map`, `filter`, `sorted` 등)에 조건식 인스턴스를 주입할 때 가장 파워풀하게 활용됩니다.

```python
# 리스트 정렬 시 두번째 요소를 기준으로 정렬하고 싶을 때
pairs = [(1, 'one'), (2, 'two'), (3, 'three'), (4, 'four')]

# lambda를 키로 활용
pairs.sort(key=lambda pair: pair[1])
print(pairs)
# [(4, 'four'), (1, 'one'), (3, 'three'), (2, 'two')]
```

## 정리
함수 안에서 탄생한 변수는 절대 바깥에서 함부로 부를 수 없다는 스코프(Scope)의 한계와 그 안전성을 뼈저리게 체감했습니다. 또한, 아주 찰나의 데이터 정렬(`sort(key=...)`) 등에서 사용하기 위해 무거운 `def` 선언문 대신 깃털처럼 가벼운 `lambda`를 사용하는 테크닉은, 여러분을 우아하고 날렵한 중급 파이썬 코더로 만들어 줄 강력한 열쇠입니다.
