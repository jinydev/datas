# 1주차 3강: 데이터 분석의 과정과 학습 로드맵

> **학습목표**: 데이터 분석의 6단계 프로세스(문제정의~평가)를 이해하고, 이를 14주차 강의 커리큘럼과 연결하여 전체적인 학습 로드맵을 파악합니다.

## 1.1.3. 데이터 분석의 과정 (Process)

데이터 분석은 맛있는 요리를 만드는 과정과 완벽하게 닮았습니다. 
좋은 재료를 구하고(수집), 깨끗하게 씻고 다듬어서(전처리), 맛을 보며(탐색), 최고의 레시피로 조리(모델링)하여 손님에게 대접(발표/배포)하는 것이죠.

우리는 이번 학기 **14주** 동안, 이 모든 과정을 파이썬이라는 도구로 직접 경험하게 될 것입니다.

---

### Step 1. 문제 정의 (Problem Definition)
> **"무엇을 요리할 것인가?"** (양식인가? 한식인가? 누구를 위한 요리인가?)

- **의미**: 해결하고자 하는 목표를 명확히 설정하는 단계입니다.
- **예시**: "왜 우리 쇼핑몰의 20대 방문자 매출이 지난달보다 10% 감소했을까?"
- **📅 학습 일정**:
    - **[1주차]** 데이터 분석의 기초 개념과 목적 이해

### Step 2. 데이터 수집 (Data Collection)
> **"신선한 재료 구하기"** (재래시장, 마트, 텃밭에서 재료 가져오기)

- **의미**: 분석에 필요한 데이터를 확보하는 단계입니다. DB, 로그 파일, 엑셀, 웹 크롤링 등 다양한 소스가 있습니다.
- **📅 학습 일정**:
    - **[3주차]** Numpy로 수치 데이터 생성하기
    - **[8주차]** Pandas로 엑셀/CSV 파일 불러오기

### Step 3. 데이터 전처리 (Data Preprocessing) ⭐️
> **"재료 다듬기"** (흙 털어내기, 껍질 까기, 먹기 좋은 크기로 썰기)

- **의미**: 수집한 데이터의 결측치(비어있는 값)를 채우거나, 이상치(잘못된 값)를 제거하여 분석 가능한 상태로 만드는 단계입니다.
- **중요성**: **"Garbage In, Garbage Out"** (쓰레기를 넣으면 결과도 쓰레기가 나온다). 전체 과정의 70~80% 시간을 차지합니다.
- **📅 학습 일정**:
    - **[4~6주차]** Numpy를 활용한 배열 형태 변환 및 조작
    - **[9주차]** Pandas를 활용한 데이터 필터링, 정렬, 결측치 처리

### Step 4. 탐색적 데이터 분석 (EDA, Exploratory Data Analysis)
> **"맛보기 및 간 맞추기"** (국물 맛을 보고, 소금이 더 필요한지 확인하기)

- **의미**: 그래프를 그려보거나 통계량을 확인하여 데이터의 특징과 패턴을 미리 파악하는 단계입니다.
- **📅 학습 일정**:
    - **[2주차]** Matplotlib으로 기초 시각화 (그래프 그리기)
    - **[10주차]** Pandas와 Seaborn을 활용한 고급 시각화 및 상관관계 분석

### Step 5. 모델링 (Modeling)
> **"본격적인 조리"** (굽기, 튀기기, 끓이기)

- **의미**: 머신러닝(기계학습) 알고리즘을 적용하여 데이터를 학습시키고, 미래를 예측하거나 분류하는 모델을 만드는 단계입니다.
- **📅 학습 일정**:
    - **[11주차]** 머신러닝 개요 및 선형 회귀 (Linear Regression)
    - **[12주차]** 분류 모델 (Classification) 만들기
    - **[13주차]** 군집화 (Clustering) 및 모델 성능 튜닝

### Step 6. 평가 및 적용 (Evaluation & Deployment)
> **"플레이팅 및 서빙"** (손님에게 내놓고 평가받기)

- **의미**: 만든 모델이 얼마나 정확한지 평가하고, 실제 비즈니스나 문제 해결에 적용하는 단계입니다.
- **📅 학습 일정**:
    - **[12주차]** 모델 성능 평가 지표 (정확도, 정밀도 등) 학습
    - **[14주차]** **기말 프로젝트**: 실제 데이터를 활용한 종합 분석 및 발표
