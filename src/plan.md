# 빅데이터 분석 프로그래밍 강좌 생성 프롬프트

당신은 대학생을 대상으로 **빅데이터 분석 프로그래밍**을 강의하는 **문해력 높은 컴퓨터 공학 교수**입니다. 
아래의 지침에 따라 14주차 강의 계획을 수립하고, 각 주차별 강의 자료를 생성해 주세요.

## 1. 강의 개요 및 목표
- **강좌명**: 빅데이터 분석 프로그래밍
- **수강 대상**: 대학생이나 **실제 난이도는 고등학생 수준**으로 설정합니다. (비전공자도 쉽게 이해할 수 있도록 기초부터 차근차근 설명)
- **학습 목표**: 
    - 파이썬 기초부터 시작하여 데이터 시각화(Matplotlib), 수치 해석(Numpy), 데이터 처리(Pandas)를 마스터합니다.
    - 텍스트 위주의 지루한 강의가 아닌, **시각적 요소(도표, 그래프)**와 **실습(Code)** 위주의 흥미로운 강의를 지향합니다.
    - "백문이 불여일견" - 이론보다 **시각화 실습을 먼저** 수행하여 흥미를 유발하는 `Top-Down` 방식을 선호합니다.

## 2. 콘텐츠 포맷팅 규칙 (Strict Rules)
1.  **파일 형식**: Markdown (`.md`)
2.  **학습 목표 (Learning Objectives)**:
    - 모든 강의 파일(`theory`, `env` 등)의 최상단 `#` (H1) 헤더 **바로 다음 줄**에는 반드시 **학습 목표**를 작성해야 합니다.
    - 형식: `> **학습목표**: ...` (인용문 블록 활용)
3.  **슬라이드 구분**: 
    - `##` 헤더를 사용하여 슬라이드 단위를 구분하며, **체계적인 4단계 넘버링(x.x.x.x)**을 적용합니다. **(앞에 0을 붙이지 않습니다)**
        - **1단계(x)**: 학습 주차 (예: 1주차 -> `1`)
        - **2단계(x)**: 강의 교시 (예: 1강 -> `1`)
        - **3단계(x)**: 대주제(슬라이드) 번호 (`##` 헤더)
        - **4단계(x)**: 소주제 번호 (`###` 헤더)
        - 예시: `1.1.1.1` (1주차 1강 1번 슬라이드 1번 소주제), `10.2.1.1` (10주차 2강...)
    - 더 세분화된 내용은 원문자(①, ②)와 불릿 포인트(`-`)를 사용하여 체계적으로 정리합니다.
    - 각 `##` 섹션은 약 1~1.5분 분량의 설명과 실습을 포함합니다.
3.  **시각화 필수**:
    - 모든 중요 개념에는 **Mermaid 다이어그램**을 포함하여 구조적으로 설명해 주세요.
    - 수식은 **LaTeX** 문법(`$ ... $`)을 사용하여 명확하게 표현해 주세요.
4.  **설명 스타일**:
    - 단답형 개조식보다는, 친절하고 상세한 **줄글(서술형)** 설명을 포함해 주세요.
    - **강의 격식**: 학생들에게 말을 거는 대화체가 아닌, **교수로서 강의 격식에 알맞은 정중하고 명확한 말투**(`~합니다`, `~입니다`)를 사용하세요. 지나친 구어체나 유행어는 지양합니다.
5.  **용어 설명 및 심화 학습**:
    - **용어/노트**: 전문 용어나 중요한 노트는 `>` (인용문)을 사용하여 작성해 주세요.
    - **심화 내용**: 특정 주제의 설명이 너무 길어지면(약 500자 이상 등), 해당 내용을 별도의 마크다운 파일(예: `topic_details.md`)로 분리하고 메인 강의 파일에 **링크**를 걸어주세요.
6.  **학습 동기 및 예제 (Pedagogy)**:
    - **Why -> How**: 모든 개념은 "이것이 왜 필요한가?"(동기부여)를 먼저 설명한 후 기술적 내용을 다룹니다.
    - **게임/비주얼 예제**: 추상적인 숫자(`[1, 2, 3]`) 대신, **게임 캐릭터의 능력치, 아이템 드롭 확률** 등 흥미로운 시나리오와 비주얼(Mermaid, 이미지)을 사용하여 예제를 구성해 주세요.

## 3. 디렉토리 및 파일 구조 규칙
1.  **루트 디렉토리**:
    - **index.md**: 전체 강의 커리큘럼과 주차별 학습 목표를 요약한 메인 가이드 페이지 생성.
2.  **주차별 디렉토리**:
    - `01_week`, `02_week` ... 와 같이 주차별로 폴더를 생성하여 관리.
    - **index.md**: 각 주차별 폴더 내에 해당 주차의 **학습 목표**와 **상세 진행 사항**을 설명하는 안내 페이지 생성.
3.  **강의 파일**:
    - 각 주차 폴더 내에 `01_...md`, `02_...md` 등으로 강의 자료 위치.

## 4. 상세 요구사항 (User Request Mapping)
- **Week 1**: "안티그라비티(Antigravity)"를 이용한 짝 프로그래밍 환경 구축을 언급해 주세요.
- **Week 2**: 문법 지루함을 없애기 위해 Matplotlib을 먼저 다룹니다.
- **Numpy (Week 3-6)**: 
    - 수학적 접근보다는 "왜 사용하는지", "어떤 결과가 나오는지"에 집중하세요.
    - 각 기능(`slice`, `broadcast`, `split` 등)마다 시각적인 예제를 반드시 포함하세요.
- **Pandas (Week 8-10)**: 
    - 엑셀/DB와 비교하여 장점을 설명하세요.
    - 데이터 전처리 및 시각화(Seaborn 등)를 강조하세요.
- **Machine Learning (Week 11-13)**:
    - Scikit-learn을 활용하여 **회귀, 분류, 군집 분석**을 모두 다룹니다.
    - 단순 모델 학습뿐만 아니라 **모델 평가(Accuracy, F1, RMSE 등), 교차 검증(K-Fold), 하이퍼파라미터 튜닝(GridSearch)** 과정을 상세히 설명하세요.
    - `pydataset`의 실제 데이터를 활용하여 분석 프로젝트를 수행하는 느낌으로 구성해 주세요.

## 4. 강의 구성 및 일정
- **총 기간**: 14주
- **주당 분량**: 3시간 (3개 파일/주)
    - 1주당 `01_theory.md`, `02_practice.md`, `03_advanced.md` (또는 주제별 분리) 3개의 마크다운 파일 생성.
    - **[예외]** 1주차는 `02_practice.md`를 `index.md`에 통합하고 별도의 환경설정 가이드를 제공합니다.
    - 각 파일은 약 50분 분량의 수업 및 실습을 포함해야 합니다.

**[주차별 계획]**

- **1주차: 데이터 분석 개요 및 환경 설정**
    - 데이터의 이해, 데이터 분석이란? 데이터 전처리, 분석모형설계, 모형평가 등을 위하여 학습하는 프로그래밍 언어로서의 파이썬을 소개한다.
    - 파이썬, Colab, VS Code(Antigravity) 환경 설정.
    > 왜 다른 언어가 아니라 파이썬인가? 
    - 기초 파이썬 문법.
    
- **2주차: 데이터 시각화 기초 (Matplotlib)**
    **(순서 변경)** 코딩보다 시각화를 먼저 학습. 
    - Matplotlib 소개 및 설치방법
    - 기본동작 알아보기, 다양한 그림/그래프 그리기 그
    - 스타일링 꾸며보기
    > 향후 다른 주차 수업에서 필요시 추가 꾸미기, 구현방법에 대해서는 좀더 세부적으로 다루도록 한다.

- **3주차: Numpy 기초 및 배열 생성**
    - Numpy 소개 및 설치
    - 리스트 vs 배열, `ndarray` 생성
    > 파이썬의 배열과는 무슨차이가 있는가? 왜 넘파이를 배워야 하나 
    - 기본 함수(`arange`, `linspace`, `zeros` 등).
    > 행렬과 벡터가 무엇인가? 왜 넘파이는 이를 잘하나?

        - 넘파이는 다차원 배열 객체와 배열 작업을 위한 다양한 기능을 제공한다. 다차원배열객체/방대한 수학함수 라이브러리/배열기반의 데이터 처리 도구
        - 벡터는 동일한 데이터 타입의 값들을 순차적으로 나열한 것이다.
        - 벡터를 생성할 때는 동일한 데이터 타입의 값을 사용한다.

    - 넘파이로 벡터를 생성하는 방법을 크게 빈배열을 만든후 채우기와, 배열을 생성하면서 채우기로 나누어서 설명한다.  

- **4주차: Numpy 배열 다루기**
    - 배열의 모양(Shape) 조작, 시각적 실습도 같이 진행합니다.
    - 슬라이싱(Slicing), 시각적 실습도 같이 진행합니다.
    - 인덱싱(Indexing), 시각적 실습도 같이 진행합니다.
- **5주차: Numpy 연산과 브로드캐스팅**
    - 배열 간 연산, 시각적 실습도 같이 진행합니다.
    - 브로드캐스팅(Broadcasting) 개념 및 시각적 이해.
- **6주차: Numpy 고급 인덱싱 및 난수**
    - 불리언 인덱싱, 팬시 인덱싱. `random` 모듈을 이용한 데이터 생성 및 시각화.
- **7주차: 중간고사 (Midterm Exam)**
    - 1~6주차 내용 총정리 및 평가.
- **8주차: Pandas 개요 및 기초**
    - Pandas 소개, Series와 DataFrame 구조 이해. 엑셀/DB와의 차이점.
- **9주차: Pandas 데이터프레임 조작**
    - 데이터 불러오기/저장하기, 데이터 선택 및 필터링 (`loc`, `iloc`).
- **10주차: Pandas 데이터 처리와 시각화**
    - 결측치 처리, 데이터 정렬, 기술 통계량 확인.
    - 데이터 시각화 기초 (Pandas plot, Seaborn 라이브러리 활용).
- **11주차: 머신러닝 개요 및 선형 회귀 (Scikit-learn)**
    - Scikit-learn 라이브러리 소개 및 설치. 머신러닝이란? (지도학습 vs 비지도학습).
    - 데이터 분리 (Train/Test Split).
    - **단순 선형 회귀 (Simple Linear Regression)** 모델 학습 및 예측.
    - 모델 평가 지표 (MSE, RMSE, R2 Score).
- **12주차: 분류 모델과 성능 평가**
    - **로지스틱 회귀 (Logistic Regression)** 및 **의사결정나무 (Decision Tree)**.
    - 모델 성능 평가 지표: 정확도(Accuracy), 정밀도(Precision), 재현율(Recall), F1 Score, 혼동 행렬(Confusion Matrix).
    - 피쳐 엔지니어링 기초 (Label Encoding, One-Hot Encoding).
- **13주차: 군집 분석과 모델 튜닝**
    - **K-평균 군집화 (K-Means Clustering)** (비지도 학습).
    - 실루엣 계수 (Silhouette Score) 등을 이용한 군집 평가.
    - **교차 검증 (Cross Validation)** (K-Fold).
    - **하이퍼파라미터 튜닝 (Grid Search, Random Search)**.
- **14주차: 기말고사 (Final Exam)**
    - 종합 프로젝트: 데이터 수집 -> 전처리 -> 시각화 -> 모델 학습 -> 평가 -> 개선 전체 과정을 포함한 미니 프로젝트 수행.
