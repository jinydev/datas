## 2.1.1. 자료구조 더 깊게 파기 (Data Structures)

프로그래밍에서 데이터는 혼자 존재하기보다 여러 개가 뭉쳐서 존재할 때가 많습니다. (예: 학생 명단, 일주일치 날씨, 장바구니 목록 등)
이런 데이터들을 효율적으로 담고 관리하기 위한 바구니를 **자료구조(Data Structure)**라고 합니다.

### 2.1.1.1. 리스트 (List): 순서가 있는 가방
리스트는 가장 많이 사용되는 자료구조로, **순서(Order)**를 가지고 데이터를 나열하여 저장합니다. 대괄호 `[]`를 사용하여 만듭니다.

**특징:**
*   순서가 있다 (인덱싱 가능).
*   내용을 바꿀 수 있다 (수정, 추가, 삭제 가능).
*   어떤 자료형이든 담을 수 있다 (숫자, 문자, 리스트 등).

```python
# 아이템 목록 생성
items = ["포션", "장검", "방패"]

# 1. 데이터 접근 (인덱싱 & 슬라이싱)
# 파이썬의 순서는 0부터 시작합니다.
print(items[0])   # 첫 번째 아이템: 포션
print(items[1])   # 두 번째 아이템: 장검
print(items[-1])  # 뒤에서 첫 번째: 방패

# 2. 데이터 추가 (append)
items.append("지도") # 리스트의 맨 끝에 '지도'를 추가합니다.
print(items) # ['포션', '장검', '방패', '지도']

# 3. 데이터 삭제 (remove)
items.remove("포션") # '포션'이라는 값을 찾아 지웁니다.
print(items) # ['장검', '방패', '지도']

# 4. 데이터 길이 확인 (len)
print(len(items)) # 3
```

### 2.1.1.2. 딕셔너리 (Dictionary): 이름표가 붙은 금고
리스트가 번호(0, 1, 2...)로 데이터를 찾았다면, 딕셔너리는 **이름(Key)**으로 데이터(Value)를 찾습니다. 중괄호 `{}`를 사용하며, `Key: Value` 쌍으로 이루어져 있습니다.

**특징:**
*   순서보다는 **키(Key)**가 중요하다.
*   키(Key)는 중복될 수 없다. (주민등록번호나 ID처럼 유일해야 함)

```python
# 캐릭터 정보 생성 (Key: Value)
profile = {
    "name": "Antigravity",
    "level": 10,
    "hp": 100
}

# 1. 데이터 접근 (Key로 Value 찾기)
print(profile["name"]) # 결과: Antigravity
print(profile["hp"])   # 결과: 100

# 2. 데이터 수정
profile["level"] = 11 # level을 11로 변경
print(f"레벨 업! 현재 레벨: {profile['level']}")

# 3. 새로운 데이터 추가
profile["job"] = "Wizard" # job이라는 Key와 Wizard라는 Value 추가
print(profile)
```

## 2.1.2. 제어문 (Control Flow)

지금까지는 코드가 위에서 아래로 순서대로 실행되었습니다. 하지만 실제 프로그램은 상황에 따라 다른 일을 하거나, 같은 일을 반복해야 합니다. 이를 제어하는 것이 **제어문**입니다.

### 2.1.2.1. 조건문 (if-elif-else): "만약에..."
주어진 조건이 **참(True)**인지 **거짓(False)**인지에 따라 실행할 코드를 결정합니다.
*   **들여쓰기(Indentation)**가 매우 중요합니다! (보통 스페이스 4칸 또는 탭 1번)

```python
hp = 30

# 조건 1: 체력이 20 이하니?
if hp <= 20:
    print("⚠️ 위험! 포션을 사용하세요!") # 참일 때 실행
    
# 조건 2: (위 조건이 아니고) 체력이 50 이하니?
elif hp <= 50:
    print("주의! 체력을 관리하세요.")   # 조건 1은 거짓, 조건 2는 참일 때 실행
    
# 그 외 모든 경우
else:
    print("상태 양호. 전투를 계속합니다.") # 위 모든 조건이 거짓일 때 실행
```

### 2.1.2.2. 반복문 (for loop): "계속해..."
특정 코드를 반복해서 실행하고 싶을 때 사용합니다. 리스트의 모든 아이템을 하나씩 꺼내보거나, 숫자를 세면서 반복할 수 있습니다.

**1. 리스트 순회 (Items in List)**
리스트 안에 있는 아이템을 하나씩 꺼내서 변수(`monster`)에 담아 반복합니다.

```python
monsters = ["Slime", "Orc", "Dragon"]

for monster in monsters:
    print(f"야생의 {monster}가 나타났다!")
    # 첫 번째 반복: monster = "Slime"
    # 두 번째 반복: monster = "Orc"
    # 세 번째 반복: monster = "Dragon"
```

**2. 숫자 범위 반복 (range)**
숫자를 만들어주는 `range(시작, 끝, 간격)` 함수와 함께 사용합니다.
*   주의: **끝 번호는 포함되지 않습니다!** (1, 6)이면 1부터 5까지입니다.

```python
# 1부터 5까지 카운트다운
for i in range(1, 6):
    print(f"카운트다운: {i}")

# Tip: 0부터 9까지 반복하려면 range(10)이라고만 써도 됩니다.
```

## 2.1.3. 종합 실습: 텍스트 RPG 전투 시스템

지금까지 배운 리스트, 반복문, 조건문을 모두 사용하여 간단한 게임 로직을 만들어 봅시다.

**시나리오:**
1.  플레이어의 체력은 100입니다.
2.  몬스터 리스트에는 슬라임과 늑대가 있습니다.
3.  각 몬스터를 순서대로 만납니다. (반복문)
4.  몬스터에게 랜덤하게 공격을 받습니다. (변수, 연산)
5.  체력이 0 이하가 되면 게임 오버입니다. (조건문, break)
6.  살아남으면 승리 메시지를 출력합니다.

```python
import random # 랜덤 숫자를 뽑기 위한 모듈

player_hp = 100
monsters = ["Slime", "Wolf", "Dragon"]

print("⚔️ 모험을 시작합니다!")

for monster in monsters:
    print(f"\n--- {monster}와의 전투 시작! ---")
    
    # 10에서 50 사이의 랜덤 데미지 생성
    damage = random.randint(10, 50) 
    player_hp = player_hp - damage
    
    print(f"으악! {monster}에게 {damage}의 데미지를 입었습니다.")
    
    # 체력 체크
    if player_hp <= 0:
        print(f"체력이 {player_hp}이 되었습니다.")
        print("💀 Game Over... 눈앞이 캄캄해집니다.")
        break # 반복문을 즉시 종료하고 빠져나갑니다.
    else:
        print(f"휴... 남은 체력: {player_hp}")

# 반복문이 정상적으로 끝났을 때만 실행 (break로 안 끊겼을 때)
# 다만 여기서는 간단하게 체력으로 확인합니다.
if player_hp > 0:
    print("\n🎉 축하합니다! 모든 몬스터를 물리치고 승리했습니다!")
```
