# 4.3.2 배열과 배열의 연산

**[비유로 이해하기: 가내수공업 vs 스마트 공장 (Element-wise)]**
- **파이썬 반복문 (for 루프)**: 장인이 한땀 한땀 바느질하듯 처리합니다. 코드가 길고 속도가 느립니다.
- **배열 연산**: 공장 기계가 한 번에 `쿵!` 찍어내듯 같은 위치의 요소끼리 자동으로 초고속 계산됩니다. 내부적으로 C언어에 최적화되어 파이썬 반복문 대비 어마어마한 속도 향상 효과가 있습니다.

다음 코드로 2행 3열의 2차원 배열 `a`와 `b`를 생성한다.

```python
a = np.arange(1, 7).reshape(2, 3)
a
```
**출력:**
```
array([[1, 2, 3],
       [4, 5, 6]])
```

```python
b = np.full_like(a, [1, 2, 3])
b
```
**출력:**
```
array([[1, 2, 3],
       [1, 2, 3]])
```

모양이 같은 두 배열의 사칙 연산도 같은 위치의 원소끼리 연산이 수행된 배열이 반환된다.

```python
print(a + b)
print(a - b)
print(a * b)
print(a / b)
```
**출력:**
```
[[2 4 6]
 [5 7 9]]

[[0 0 0]
 [3 3 3]]

[[ 1  4  9]
 [ 4 10 18]]

[[1.  1.  1. ]
 [4.  2.5 2. ]]
```

배열 연산에서도 `+=`, `-=`, `*=`, `/=` 등의 복합 대입 연산자도 지원한다. 다음 변수 `a`는 `int32`형이며, 변수 `b`는 `dtype`이 `float`이다.

```python
a = np.arange(4)
a
```
**출력:**
```
array([0, 1, 2, 3])
```

```python
b = np.arange(0.5, 4, 1)
b
```
**출력:**
```
array([0.5, 1.5, 2.5, 3.5])
```

다음 연산 `a += b`에서 오류가 발생한다. `a`의 원소 자료형은 `int`인데 저장할 값은 `float`이기 때문에 발생하는 문제이다. 이러한 자료형에 유의하자. 물론 `b += a`는 문제없이 수행된다.

```python
a += b
```
**오류:**
```
UFuncTypeError: Cannot cast ufunc 'add' output from dtype('float64') to dtype('int32') with casting rule 'same_kind'
```

```python
b += a
b
```
**출력:**
```
array([0.5, 2.5, 4.5, 6.5])
```

앞 배열의 행과 뒤 배열의 열이 같아야 수행할 수 있는 배열의 곱 연산자는 `@`이다. 다음은 (3, 2) 모양 배열과 (2, 1) 모양 배열의 연산 방법과 결과를 나타낸다. 곱의 결과는 (3, 1) 3행 1열의 2차원 배열이 된다.

다음 두 배열 (2, 3) 모양의 `a`와 (3, 2) 모양의 `b`가 있다. 배열 `a`의 열과 배열 `b`의 행이 모두 3이므로 곱 수행이 가능하며, 결과는 (2, 2)이다.

```python
a = np.full((2, 3), [1, 2, 3])
a
```
**출력:**
```
array([[1, 2, 3],
       [1, 2, 3]])
```

```python
b = np.full((3, 2), [2, 1])
b
```
**출력:**
```
array([[2, 1],
       [2, 1],
       [2, 1]])
```

두 배열 `a`, `b`의 곱 연산은 `@` 연산자, 또는 `ndarray`의 `a.dot(b)` 함수 호출로는 가능하다.

```python
a @ b
```
**출력:**
```
array([[12,  6],
       [12,  6]])
```

```python
a.dot(b)
```
**출력:**
```
array([[12,  6],
       [12,  6]])
```

일반적으로 배열의 곱은 교환법칙이 성립하지 않는다. 그런데, 위 `b`와 `a`의 `b.dot(a)` 연산에서 (3, 2) @ (2, 3)이므로 곱의 결과 (3, 3) 모양의 결과가 성립한다.

```python
b.dot(a)
```
**출력:**
```
array([[3, 6, 9],
       [3, 6, 9],
       [3, 6, 9]])
```
