# 4.3.4 브로드캐스팅

## 4.3.4 브로드캐스팅 개요

**[수학적 의미: 차원 복제와 공간 확장 (Sweeping)]**
브로드캐스팅(Broadcasting)은 Numpy가 제공하는 가장 강력하고 마법 같은 연산 규칙입니다. 
수학에서 1차원 선분(Vector)을 y축 방향으로 똑같이 쭈욱 밀고 나가면 2차원 면적(Matrix)이 되는 기하학적 원리를 생각해 보세요. 
크기가 서로 다른 배열끼리 연산을 시도할 때, Numpy는 에러를 뿜는 대신 **"부족한 차원을 복제하여 상대방의 거대한 차원 공간에 억지로 맞춘 뒤 연산"**을 시도합니다.

**[비유로 이해하기: 광역 파티 버프 시전]**
게임에서 파티장 1명이 전체 파티원 5명에게 **"모두 공격력 +10!"**이라는 단일 스킬(광역 버프)을 겁니다. 스칼라 값(`10`) 하나가 마법처럼 5명 크기의 배열로 쫙 늘어나(Broadcast) 동시에 덧셈 연산이 수행되는 직관적인 원리입니다.

이미 보았듯이 NumPy 연산은 일반적으로 요소 단위(Element-wise)로 기계 장치를 모방해 수행됩니다. 가장 간단하고 안전한 연산의 전제조건은 당연히 "두 타일의 가로세로 규격(Shape)이 완벽히 일치해야 한다"는 것입니다.

```python
from numpy import array

a = array([3, 4, 5])
b = array([8, 7, 6])
a + b
```
**출력:**
```
array([11, 11, 11])
```

두 배열의 모양이 애매하게 다르면 다음 코드처럼 `ValueError`가 발생한다.

```python
a = array([3, 4, 5, 7])
b = array([8, 7, 6])
a * b
```
**오류:**
```
ValueError: operands could not be broadcast together with shapes (4,) (3,)
```
