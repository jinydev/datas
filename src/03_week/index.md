# 3주차: Numpy 기초 및 배열 생성

> **학습 목표 (Learning Objectives)**
> 1.  **Numpy의 필요성**: 파이썬 리스트와 비교하여 Numpy 배열이 왜 빠르고 효율적인지 이해합니다.
> 2.  **벡터와 행렬**: 데이터 분석의 기본 단위인 스칼라, 벡터, 행렬의 개념을 시각적으로 익힙니다.
> 3.  **배열 생성**: `ndarray`를 생성하는 다양한 방법(`zeros`, `ones`, `arange` 등)을 실습합니다.

## 강의 내용 (Course Content)

### [Part 1. 파이썬 자료구조 심화 (Python Data Structures Deep Dive)](./01_python_list.html)
> 데이터 분석의 기초가 되는 리스트, 튜플, 딕셔너리의 핵심 기능을 깊이 있게 다룹니다.

#### [01. 파이썬 리스트 심화](./01_python_list.html)
- **메모리 구조**: 참조(Reference)와 복사(Copy).
- **고급 기능**: 리스트 컴프리헨션, `zip`, `enumerate`.
- **필수 메서드**: `append`, `extend`, `pop`, `remove` 완벽 정리.

#### [02. 파이썬 튜플 심화](./02_python_tuple.html)
- **불변성(Immutable)**: 데이터 무결성을 지키는 약속.
- **패킹과 언패킹**: 우아한 파이썬 코딩 기법.
- **효율성**: 리스트보다 가볍고 빠른 자료구조.

#### [03. 파이썬 딕셔너리 심화](./03_python_dict.html)
- **해시 테이블(Hash Table)**: O(1) 검색 속도의 비밀.
- **안전한 코딩**: `get()` 메서드와 예외 처리.
- **딕셔너리 컴프리헨션**: 데이터를 빠르게 가공하는 기술.

### [Part 2. Numpy 핵심 (Numpy Essentials)](./04_numpy_intro.html)
> 데이터를 숫자로 바라보는 새로운 관점, 벡터와 행렬의 세계로 초대합니다.

#### [04. Numpy란 무엇인가?](./04_numpy_intro.html)
- 리스트(List) vs 배열(Array): 속도와 효율성 비교.
- **이사짐 트럭 비유**: 빠른 데이터 운송을 위한 구조적 차이.
- 왜 Numpy를 배워야 하는가? (대용량 데이터 처리의 핵심).

#### [05. 벡터와 행렬](./05_vector_matrix.html)
- **데이터의 형태**: 스칼라(0D), 벡터(1D), 행렬(2D), 텐서(3D+).
- **시각적 비유**: 기차(Vector), 계란판(Matrix), 큐브(Tensor).
- 차원(Dimension)과 모양(Shape) 이해하기.

#### [06. 배열 생성하기 1: 채우기](./06_numpy_creation.html)
- 배열 생성의 두 가지 접근: 빈 배열 만들기 vs 값으로 채우기.
- `np.zeros()`, `np.ones()`, `np.full()`: 초기화 함수들.

#### [07. 배열 생성하기 2: 범위와 간격](./07_numpy_ranges.html)
- `np.arange()`: 규칙적인 수열 만들기.
- `np.linspace()`: 구간을 일정하게 나누기.
- `np.random`: 난수로 배열 생성하기 맛보기.

## 3.3. 정리 (Summary)

이번 주차에는 데이터 분석의 기초 체력이 되는 **파이썬 자료구조 심화**와 **Numpy의 핵심 개념**을 학습했습니다.

1.  **파이썬 자료구조 심화 (Part 1)**:
    -   **리스트(List)**: 메모리 구조와 컴프리헨션으로 효율성 높이기.
    -   **튜플(Tuple)**: 불변성을 이용한 안전한 데이터 관리.
    -   **딕셔너리(Dict)**: 해시 테이블을 이용한 초고속 검색.
2.  **Numpy 핵심 (Part 2)**:
    -   **배열(Array)**: 리스트보다 빠르고 강력한 데이터 저장소.
    -   **벡터와 행렬**: 데이터 분석을 위한 수학적 구조 시각화.
    -   **배열 생성**: `zeros`, `arange` 등으로 원하는 형태의 데이터셋 준비.

다음 주차(**Week 4**)에는 이 강력한 배열을 자유자재로 다루는 **인덱싱(Indexing), 슬라이싱(Slicing), 그리고 형태 변환(Reshape)** 기술을 연마합니다. 🧩
