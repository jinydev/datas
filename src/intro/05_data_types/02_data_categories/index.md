# 1.5.2 데이터를 구조화해야 하는 이유 정리
왜 이렇게 귀찮게 데이터의 종류를 다 외우고 구분해야 할까요? 왜냐하면 우리가 파이썬을 이용해 어떤 막대그래프를 그릴지, 아니면 꺾은선 그래프 모양을 그릴지는 전적으로 데이터의 '종류'에 따라 결정되기 때문입니다. 



![그래프 분류](../img/missing_value_puzzle.png)


## 데이터 종류와 결측치(빈값)의 처리
실제 세상의 빅데이터는 완벽하지 않습니다. 

나이를 안 적은 사람, 성별을 응답하지 않은 사람처럼 데이터에 빵꾸(결측치)가 뚫려 있습니다. 나이가 빵꾸났다면 '전체 평균 나이'로 채워 넣으면 되지만, 성별에 빵꾸가 났다고 '평균 성별 1.5'를 채울 순 없습니다. 가장 많이 등장한 성별(최빈값)로 채워야만 합니다. 데이터의 성격을 정확히 알아야만 이런 적절한 조치를 취할 수 있습니다.


![데이터 개념](../img/data_tree_classification.png)



### 빅데이터의 3V 특성
이제 데이터 하나의 크기에서 거대한 빅데이터로 시야를 넓혀봅시다. 

빅데이터라 불리기 위해서는 3가지의 V조건이 충족되어야 합니다. 첫 번째는 방대한 양(Volume)입니다. 테라바이트(TB), 페타바이트(PB) 수준의 압도적인 크기를 말합니다.
